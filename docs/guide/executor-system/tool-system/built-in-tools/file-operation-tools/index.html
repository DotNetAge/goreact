    文件操作工具¶

    简介¶

    本文件操作工具参考文档聚焦于三个核心工具：Read（读取）、Write（写入）、Edit（编辑）。文档从架构与职责、参数与输入输出、安全限制、错误处理、最佳实践与安全注意事项等维度进行系统阐述，并结合实际应用场景与流程图帮助读者快速掌握使用方法。

    项目结构¶

    文件操作工具位于工具库的内置模块中，遵循统一的工具接口规范，便于被上层思考器与执行器调用。其主要文件包括： - Read：文件读取与行范围切片 - Write：文件写入与追加，含目录创建与权限控制 - Edit：多位置精确编辑，基于“旧文本→新文本”的替换策略 - 公共校验与安全：参数校验、路径清理与特殊文件保护 - 工具接口与管理器：定义工具契约与注册发现

    核心组件¶

    • Read：读取文件内容，支持指定起止行；对大文件有上限限制；返回统计信息与带行号的内容。
    • Write：写入或追加内容，自动创建目录；对敏感文件进行安全检查；返回写入模式、字节数与最终大小。
    • Edit：按顺序在文件中进行多处“旧→新”替换；若任一处找不到旧文本则回滚并报错；返回编辑数量、尺寸变化与各区域统计。

    架构总览¶

    工具通过统一接口暴露能力，执行时由上层引擎/思考器传入参数映射，工具内部完成参数校验、安全检查、文件操作与结果组装。

    详细组件分析¶

    Read 工具¶

    • 职责与特性
    • 读取文件内容，可选起止行范围；自动统计总行数与读取行数；为每行附加行号。
    • 对文件大小进行上限控制，避免过大文件导致内存压力。
    • 对非文件路径（如目录）进行拒绝。
    • 关键参数
    • path：必填，字符串，目标文件路径。
    • start_line：可选，数值，起始行（从1开始）。
    • end_line：可选，数值，结束行（从1开始）。
    • 输出结构
    • success：布尔，是否成功。
    • path：字符串，文件路径。
    • size_bytes：整数，文件大小（字节）。
    • lines_read：整数，本次读取的行数。
    • total_lines：整数，文件总行数。
    • content：字符串，带行号的文本内容。
    • startline/endline：当指定范围时返回。
    • 安全与限制
    • 文件大小上限（默认10MB）。
    • 路径必须指向文件，目录将被拒绝。
    • 错误处理
    • 路径不存在、打开失败、扫描失败、参数缺失或类型不符均会返回错误。
    • 使用场景
    • 日志片段提取、配置文件片段读取、代码片段审阅等。

    依赖分析¶

    • 工具接口与管理器
    • 工具通过统一接口暴露能力，便于注册与发现。
    • 管理器负责工具注册、检索与动态筛选。
    • 文件操作工具的内部依赖
    • 三个工具均依赖公共校验与安全函数，保证一致性。
    • Read/Write/Edits 均使用标准库进行文件读写与路径处理。

    性能考虑¶

    • Read
    • 支持行范围读取，避免一次性加载大文件。
    • 默认最大文件大小限制为 10MB，降低内存占用。
    • Write
    • 追加模式与覆盖模式分别适配不同场景；自动创建目录减少多次 IO。
    • Edit
    • 顺序替换，仅替换首次出现，避免重复扫描；适合批量小范围修改。
    • 通用建议
    • 批量修改优先使用 Edit 一次性提交，减少多次读写。
    • 避免频繁的 Read/Write 循环，尽量合并操作。

    故障排查指南¶

    • 常见错误与定位
    • 参数缺失或类型不符：检查 path、content、edits 等是否正确传递。
    • 文件不存在或路径为目录：确认路径指向文件而非目录。
    • 文件过大：Read 限制为 10MB；必要时拆分读取或改用流式处理。
    • 旧文本未找到：Edit 的每条替换必须能在当前内容中找到。
    • 安全限制：对 passwd、shadow、sudoers 等敏感文件的访问会被拒绝。
    • 目录创建失败或写入失败：检查权限与磁盘空间。
    • 排查步骤
    • 先用 Read 确认文件存在与大小。
    • 再用 Write/ Edit 验证权限与模式。
    • 若失败，查看返回的错误信息并对照上述原因逐项排除。

    结论¶

    Read/Write/Edit 三工具围绕“读取、写入、编辑”三大核心场景，提供了参数严谨、安全可控、输出清晰的能力封装。通过统一接口与公共校验，它们既满足日常开发与运维需求，又在安全与性能方面给出明确约束与最佳实践建议。建议在自动化流程中优先组合使用，以提升效率与稳定性。

    附录¶

    参数与输入输出速查¶

    • Read
    • 输入：path（必填），startline（可选），endline（可选）
    • 输出：success、path、sizebytes、linesread、totallines、content、startline（可选）、end_line（可选）
    • Write
    • 输入：path（必填），content（必填），append（可选，默认 false）
    • 输出：success、path、mode、byteswritten、totalsize、message
    • Edit
    • 输入：path（必填），edits（必填，数组，元素含 oldtext 与 newtext）
    • 输出：success、path、editsapplied、originalsize、newsize、sizedelta、edited_regions、message
