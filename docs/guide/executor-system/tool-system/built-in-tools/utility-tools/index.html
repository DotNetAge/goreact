    实用辅助工具¶

    简介¶

    本文件为“实用辅助工具”的参考文档，聚焦于工具接口与内置工具集的设计与使用。内容涵盖： - 工具接口与管理器的职责边界与安全等级 - 内置工具的功能说明、输入输出约定与典型用法 - 工具提供者与注册表的扩展机制 - 设计原则、复用模式与最佳实践 - 使用注意事项与性能考量

    项目结构¶

    工具相关的核心代码位于 pkg/tools 下，主要由三部分组成： - 工具抽象层：定义工具接口、安全等级、通用管理器与工具提取辅助 - 内置工具集：文件操作、搜索浏览、执行扩展等常用能力 - 工具提供者与注册表：用于集成第三方工具生态（如 MCP、LangChain 等）

    核心组件¶

    • 工具接口与安全等级
    • 工具接口定义名称、描述、安全等级与执行方法
    • 安全等级分为安全、敏感、高风险三档，用于授权与策略控制
    • 工具管理器
    • 简单内存注册表，提供按名检索与工具清单列举
    • 工具辅助
    • 输入提取：将 map 参数严格解包到结构体，便于强类型处理
    • 工具提供者与注册表
    • 抽象第三方工具生态接入点，支持初始化、发现、获取、关闭与健康检查

    架构总览¶

    工具体系采用“接口抽象 + 内置实现 + 可插拔提供者”的分层设计： - 上层业务通过工具接口与管理器进行解耦 - 内置工具覆盖文件、搜索、执行与时间日历等高频场景 - 提供者注册表统一管理外部工具生态，支持健康检查与批量发现

    工具接口与管理器¶

    • 设计要点
    • 以最小契约定义工具行为，便于与 LLM 的动作调用对接
    • 安全等级作为授权与审计依据
    • 管理器负责工具发现与检索，支持简单与语义化两种模式
    • 典型用法
    • 通过管理器注册内置工具，按意图动态筛选
    • 使用 MapTool 包装函数式工具，快速适配已有逻辑

    故障排查指南¶

    • 参数缺失或类型不匹配
    • 使用通用校验函数确保必填参数与类型正确
    • 文件访问受限
    • 检查路径清理与敏感文件限制规则
    • 搜索无结果
    • 确认包含模式、隐藏文件过滤与文件大小限制
    • Cron 表达式无效
    • 使用 validate 操作先校验表达式，再进行解析与计算
    • Bash 执行失败
    • 检查命令合法性与环境变量，关注错误输出

    结论¶

    该工具体系以清晰的接口抽象与模块化设计，提供了覆盖文件、搜索、执行与时间日历等场景的实用工具集，并通过提供者与注册表机制实现对第三方生态的可插拔扩展。遵循安全等级与通用校验，有助于在保证安全的前提下提升开发效率与可维护性。

    最佳实践¶

    • 在工具实现中始终进行参数校验与类型转换
    • 对高风险工具（如 Bash）严格限制使用范围与授权策略
    • 使用管理器集中注册与检索工具，便于按上下文动态选择
    • 通过提供者注册表统一接入第三方工具生态，保持接口一致性
