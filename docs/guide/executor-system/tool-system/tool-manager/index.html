    工具管理器¶

    简介¶

    本文件面向 GoReAct 工具管理器，系统性阐述工具接口与管理器设计、SimpleManager 的内存注册表实现、工具发现与检索机制，并给出使用示例与扩展思路（如基于向量数据库的工具 RAG）。文档同时兼顾非技术读者的理解需求，采用渐进式讲解与可视化图示。

    项目结构¶

    工具管理器位于 pkg/tools 子模块，核心接口与默认实现如下： - 工具接口与默认实现：pkg/tools/tool.go - 内置工具集合：pkg/tools/builtin/* - 工具提供者注册表：pkg/tools/provider/provider.go - 示例与测试：examples/crontool/cronexample.go、pkg/tools/builtin/builtin_test.go

    核心组件¶

    • 工具接口 Tool：定义唯一名称、描述、安全等级、执行方法。
    • 管理器接口 Manager：定义按名检索与按意图列出工具的能力。
    • SimpleManager：内存注册表，默认实现，不执行语义检索。
    • MapTool：将任意函数包装为 Tool，便于快速实现自定义工具。
    • 内置工具：Calculator、Bash 等，体现安全等级与输入校验。
    • Provider 接口与注册表：用于集成外部工具系统（如 MCP、LangChain 等）。

    架构总览¶

    工具管理器在 GoReAct 中承担“工具发现与检索”的职责，与思考器、执行器协同工作。典型流程： - 用户意图触发后，思考器根据上下文生成动作计划； - 执行器通过工具管理器获取具体工具； - 工具执行完成后返回结果，进入记忆与演化阶段。

    工具接口与安全等级¶

    • 工具接口 Tool 规定：
    • 唯一名称（用于调用）
    • 描述（用于提示与授权）
    • 安全等级（用于授权控制）
    • Execute 方法（接收原始输入映射，返回结果或错误）
    • 安全等级：
    • LevelSafe：纯查询，无副作用
    • LevelSensitive：有限写入（如特定目录）
    • LevelHighRisk：高风险（如任意命令执行）

    故障排查指南¶

    • 工具未找到：确认名称是否与 Tool.Name() 完全一致
    • 输入参数缺失或类型不符：检查 ExtractInput 与工具内部参数校验
    • 高风险工具被拒绝：确认安全等级与授权策略
    • Provider 不可用：检查健康检查与初始化配置

    结论¶

    GoReAct 的工具管理器以简洁接口与内存实现为基础，既满足快速开发，又为后续引入语义检索与外部工具集成预留空间。通过内置工具的安全实践与 Provider 抽象，可在保证安全的前提下灵活扩展能力边界。

    使用示例：注册自定义工具与动态获取工具列表¶

    • 注册自定义工具
    • 使用 MapTool 将函数包装为 Tool，然后通过 SimpleManager.Register 注册
    • 或实现 Tool 接口的自定义类型并通过 SimpleManager.Register 注册
    • 动态获取工具列表
    • 调用 SimpleManager.ListAvailableTools(ctx, intent)，当前返回全部工具
    • 未来可基于 intent 进行语义检索，返回 Top-K 工具
