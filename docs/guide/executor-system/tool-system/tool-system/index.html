    工具系统¶

    简介¶

    本文件系统性阐述 GoReAct 的工具系统，涵盖工具接口设计与实现、工具管理器、内置工具族（计算器、日期时间、Bash、文件操作、搜索、列表、Cron）、MCP 提供商集成方式，并给出自定义工具开发、注册与调用的实践路径、测试策略与性能优化建议。目标是帮助读者快速理解并安全高效地扩展与使用工具体系。

    项目结构¶

    工具系统主要由以下模块构成： - 工具接口与管理器：定义统一的 Tool 接口、安全等级、通用管理器与简单内存管理器 - 内置工具族：文件操作（读/写/编辑）、搜索（Glob/Grep/LS）、执行扩展（Bash/Calculator/DateTime/Cron） - 提供商与集成：Provider 接口与注册表，以及 MCP（Model Context Protocol）提供商实现 - 示例与用法：Cron 工具示例展示如何直接调用工具

    核心组件¶

    • 工具接口 Tool
    • 方法：Name()、Description()、SecurityLevel()、Execute(ctx, input)
    • 参数规范：Execute 接收 map[string]any 类型的输入；可通过 ExtractInput 辅助解析为严格结构体
    • 返回值：任意类型结果或错误
    • 安全等级 SecurityLevel
    • LevelSafe：纯查询、无副作用（如 Read、Calculator）
    • LevelSensitive：有限写入（如 Write 指定目录）
    • LevelHighRisk：高风险、任意执行（如 Bash、SQL Drop）
    • 管理器 Manager
    • GetTool(name string)：按名称检索工具
    • ListAvailableTools(ctx, intent)：根据意图动态返回相关工具（可扩展为语义检索）
    • 简单管理器 SimpleManager
    • 基于内存映射的工具注册与查询
    • 支持批量 Register([...])

    架构总览¶

    工具系统采用“统一接口 + 可插拔提供商”的架构。内置工具通过 SimpleManager 注册，也可通过 Provider/Registry 从外部系统（如 MCP）动态发现与使用。调用链路清晰：Actor/Thinker 通过 Manager 获取工具，再调用 Tool.Execute 执行具体动作。

    工具接口与通用工具¶

    • MapTool：将普通函数封装为 Tool，便于快速实现简单工具
    • ExtractInput：将 map 输入序列化后再反序列化为目标结构体，保证参数类型安全

    依赖分析¶

    • 内置工具依赖工具接口与通用辅助函数（如参数校验、文件安全检查）
    • MCP 提供商依赖 HTTP 客户端与工具接口
    • 管理器与提供商之间通过接口解耦，便于扩展

    性能考虑¶

    • IO 限制：文件读写、目录遍历、正则扫描应设置合理上限（如文件大小、搜索深度、匹配数量）
    • 并发与超时：外部调用（如 MCP）需设置合理超时，避免阻塞
    • 结果缓存：对重复的静态查询（如目录列表）可考虑短期缓存
    • 参数校验前置：在 Execute 内尽早校验参数，减少无效执行
    • 安全优先：高风险工具（如 Bash）应严格授权与审计

    故障排查指南¶

    • 参数缺失或类型不匹配：检查调用侧传参键名与类型，必要时使用 ExtractInput 辅助
    • 文件访问失败：确认路径存在、非目录、未超出大小限制、无敏感文件名
    • Bash 执行异常：检查命令合法性、权限与环境变量
    • MCP 连接失败：核对 server_url、API Key、网络连通性与健康检查返回码
    • 工具未找到：确认已通过 SimpleManager.Register 或 Provider/Registry 正确注册

    结论¶

    GoReAct 工具系统以简洁统一的接口为核心，结合内置工具族与外部提供商（MCP）实现了从安全查询到高风险执行的全栈能力。通过 SimpleManager 与 Provider/Registry，系统既可满足快速集成，又具备良好的扩展性。建议在生产环境中强化参数校验、安全授权与监控告警，以获得稳定可靠的工具执行体验。

    附录¶

    自定义工具开发步骤¶

    • 实现 Tool 接口：定义 Name、Description、SecurityLevel、Execute
    • 参数解析：使用 ExtractInput 将 map 转为目标结构体
    • 安全与边界：在 Execute 内进行参数与资源边界检查
    • 注册与调用：通过 SimpleManager.Register 注册，或通过 Provider/Registry 集成外部工具
