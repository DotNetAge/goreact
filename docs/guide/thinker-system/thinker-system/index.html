    思考器系统¶

    简介¶

    本文件面向 GoReAct 思考器系统，系统性阐述 Thinker 接口定义与默认实现、模式解析（/plan 与 /specs）、Codeword 处理机制、提示协议系统的 Fluent API、与记忆系统与工具管理器的集成关系，并提供配置与使用示例路径、常见问题与性能优化建议。目标是帮助读者从概念到落地全面掌握思考器的设计与实践。

    项目结构¶

    思考器位于 pkg/thinker 下，围绕 Thinker 接口与 defaultThinker 实现展开；提示协议由 pkg/prompt 子系统提供 Fluent API、Token 计数与上下文压缩；工具与记忆分别由 pkg/tools 与 pkg/memory 提供；执行步骤封装于 pkg/steps。

    核心组件¶

    • Thinker 接口：定义思考入口，接收 PipelineContext 并在其中更新决策、轨迹与最终结果。
    • Default Thinker：默认实现，包含意图与模式解析、工具发现、提示合成、上下文压缩、LLM 调用、输出解析与反思。
    • 解析器：将 LLM 输出解析为 Trace 或最终答案；支持 /plan 与 /specs 模式下的结构化解析。
    • 提示协议：Fluent API 构建系统/用户提示，支持变量替换、历史格式化、Few-Shot、Token 计数与滑动窗口压缩。
    • 工具与记忆：工具管理器提供可用工具集合；记忆管理器提供工作记忆与语义检索注入提示。
    • 执行步骤：将 Thinker 包装为管道步骤，接入统一执行管线。

    架构总览¶

    思考器在 ReAct 循环中承担“推理-规划-决策”的中枢职责，贯穿模式解析、提示合成、上下文压缩、LLM 推理与输出解析等子管线。

    详细组件分析¶

    Thinker 接口与默认实现¶

    • 接口职责：在每次 ReAct 步骤中评估当前状态，决定下一步动作或结束任务，并将结果写入 PipelineContext。
    • 默认实现关键流程：
    • 自维护命令短路：支持 /clear 清理上下文、/compress 强制压缩。
    • 模式解析：识别 /plan（规划模式）、/specs（规格模式）与默认 ReAct 模式。
    • 工具发现：通过工具管理器列出当前可用工具。
    • 提示合成：使用 Fluent 构建器装配系统模板、任务、工具描述、历史与记忆变量。
    • 上下文压缩：针对历史对话启用滑动窗口策略，必要时可强制更激进压缩。
    • LLM 执行：通过流式聊天接口获取响应，同时回调记录令牌用量与实时思考流。
    • 输出解析：根据模式调用不同解析逻辑，失败时自动插入格式反思轨迹。

    依赖分析¶

    • 组件内聚与耦合：
    • Default Thinker 与提示协议、解析器、工具与记忆紧密耦合，形成高内聚的思考子管线。
    • Fluent 构建器对 Token 计数与压缩策略为可插拔依赖，便于扩展不同计数算法与压缩策略。
    • 外部依赖：
    • gochat 客户端用于流式推理与事件回调。
    • 工具与记忆接口抽象良好，便于替换实现。

    性能考量¶

    • 上下文压缩：
    • 使用滑动窗口策略保留最近 N 轮，避免历史过长导致超限。
    • 在 /compress 强制模式下可进一步收紧窗口，快速瘦身。
    • 令牌估算：
    • 混合语言估算器对中英混排更准确；可配合缓存计数器减少重复计算。
    • LLM 调用：
    • 通过 Usage 回调统计 Prompt/Completion 令牌，辅助成本控制。
    • 工具与记忆：
    • 工具列表与记忆注入应适度，避免过度膨胀提示模板。

    故障排查指南¶

    • 输出格式错误：
    • 当 LLM 输出不符合 ReAct 格式时，解析器会报错；思考器自动插入“格式错误反思”轨迹，引导模型纠正。
    • 令牌超限：
    • 若上下文过长，优先启用滑动窗口压缩；必要时使用 /compress 强制压缩。
    • 工具不可用：
    • 确认工具已注册且描述正确；检查工具安全级别与授权策略。
    • 记忆为空：
    • 确认记忆模块已初始化并注入；Working/语义召回为空属正常情况。

    结论¶

    思考器系统以模块化子管线为核心，通过 Codeword 驱动的模式切换、Fluent 提示协议与上下文压缩、以及与工具与记忆的深度集成，实现了可编程、可解释、可扩展的推理-行动闭环。结合示例与最佳实践，可在生产环境中稳定落地。

    附录¶

    配置选项与使用模式¶

    • Thinker 选项：
    • WithModel：指定模型名
    • WithToolManager：注入工具管理器
    • WithMemoryBank：注入记忆银行
    • WithSystemPrompt：覆盖默认系统提示模板
    • 使用模式：
    • /plan：触发任务分解，输出结构化步骤
    • /specs：触发规格分析，输出约束与需求
    • /clear：清理上下文
    • /compress：强制压缩上下文
