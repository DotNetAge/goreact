    模式解析系统¶

    简介¶

    本文件面向 GoReAct 模式解析系统，聚焦两类“思维暗语”模式：/plan（规划模式）与/specs（规格模式）。文档深入解释模式识别机制、参数提取与验证规则，阐述 Codeword（命令词）处理流程，说明任务编译器（计划解析器）的工作流程（语法分析、语义检查、结构化输出），并给出与思考器核心逻辑的集成关系、常见解析错误与调试技巧。

    项目结构¶

    围绕模式解析系统的关键目录与文件： - 思考器与模式解析：pkg/thinker - 解析器：pkg/thinker/parser/parser.go - 思考器默认实现：pkg/thinker/defaultthinker.go - 系统提示词模板：pkg/thinker/prompt/system.go - Thinker 设计说明：pkg/thinker/README.md - 引擎与流水线：pkg/engine/reactor.go - 核心数据类型：pkg/core/types.go - 工具与安全等级：pkg/tools/tool.go - 观察器：pkg/observer/defaultobserver.go - 示例与测试：examples/* 与 examples/agent_test.go

    核心组件¶

    • 模式识别与 Codeword 处理
    • 在默认思考器中，通过输入前缀判断模式：/plan、/specs 或默认 ReAct。
    • 对应注入不同的系统提示词模板，控制 LLM 的输出风格与职责。
    • 计划解析器（ParsePlan）
    • 将自然语言规划文本解析为结构化 PlannedTask 列表，支持顺序、条件与循环三类任务形态。
    • ReAct 输出解析器（ParseLLMOutput）
    • 从 LLM 原始输出中抽取 Thought/Action/ActionInput/FinalAnswer，支持 JSON 输入清洗与错误反射。
    • 引擎与流水线
    • 反应引擎将 Thinker、Actor、Observer、Terminator 组装为循环流水线，驱动 ReAct 循环。

    详细组件分析¶

    模式识别与 Codeword 处理¶

    • 功能概述
    • 在输入以特定前缀开始时，切换思考器的模式与系统提示词模板，从而改变 LLM 的输出结构与职责。
    • 关键点
    • /plan：进入“规划模式”，要求 LLM 输出结构化阶段性目标路线图，不立即执行工具。
    • /specs：进入“规格模式”，要求 LLM 对请求与记忆体中的约束进行对比分析，输出结构化规格报告。
    • 默认：进入 ReAct 模式，输出 Thought -> Action -> ActionInput -> Observation，驱动 Actor 执行。
    • 错误处理
    • 当解析失败或格式不匹配时，自动注入一条“格式错误反思”轨迹，指导模型修正格式。

    依赖分析¶

    • 组件耦合
    • DefaultThinker 依赖 Parser、PromptTemplates、CoreTypes、ToolManager、Observer。
    • Reactor 将 Thinker、Actor、Observer、Terminator 组装为流水线。
    • 外部依赖
    • LLM 客户端（OpenAI/Ollama 兼容接口）用于流式推理。
    • 工具管理器提供工具发现与执行能力。
    • 潜在循环依赖
    • 代码采用单向依赖，未见循环导入。

    性能考虑¶

    • 上下文压缩
    • 使用滑动窗口策略限制历史长度，避免 Token 超限。
    • 提示词构建优化
    • 通过 Token 计数器预估大小，必要时启用压缩策略。
    • 流式输出
    • LLM 推理采用流式事件，实时回调 OnThoughtStream，降低感知延迟。

    故障排查指南¶

    • 常见解析错误
    • 缺少 Action/ActionInput：检查 LLM 输出是否严格遵循 ReAct 格式。
    • ActionInput 非法 JSON：确认 ActionInput 为合法 JSON 对象，避免 Markdown 包裹。
    • 未检测到 FinalAnswer：若任务已完成但未输出 FinalAnswer，需引导模型补充。
    • 自动反思与修复
    • 当解析失败时，系统会注入“格式错误反思”轨迹，包含错误原因，便于模型自我纠正。
    • 调试技巧
    • 启用思考流回调，观察每轮推理的 Thought 输出。
    • 在示例程序中设置断点，定位到 Thinker 的 Think 方法，逐步跟踪模式识别与输出解析过程。

    结论¶

    GoReAct 的模式解析系统通过“思维暗语”实现灵活的任务模式切换：/plan 专注规划、/specs 专注规格、默认 ReAct 专注执行。解析器承担了从自然语言到结构化任务与 ReAct 轨迹的核心职责，配合思考器的模式识别与提示词注入，形成稳定的推理-行动闭环。借助自动反思与调试工具，系统具备良好的可维护性与可观测性。
