    高级示例¶

    简介¶

    本文件面向高级用户与系统集成开发者，围绕 GoReAct 的三个高级示例进行深入讲解：Cron 工具示例、本地 Ollama 代理示例与云端 Qwen 代理示例。内容涵盖： - Cron 表达式解析、调度与工具集成 - 代理初始化、模型配置与任务执行流程 - 多轮对话、状态管理与错误恢复 - 高级配置、性能优化与监控集成 - 完整部署指南与常见问题排查

    项目结构¶

    本仓库采用按领域分层与按功能模块划分相结合的方式组织代码。高级示例位于 examples 目录，核心引擎与认知组件位于 pkg 目录。

    核心组件¶

    • 引擎 Reactor：装配 Thinker、Actor、Observer、Terminator 四大组件，驱动 ReAct 循环。
    • Thinker：负责提示词合成、LLM 推理、输出解析与模式切换（ReAct/规划/规范）。
    • Actor：根据上一步推理选择工具并执行，将结果写入上下文总线。
    • Observer：将 Actor 的原始结果序列化为可读文本，供 LLM 下一轮推理使用。
    • Terminator：终止条件检查（完成标志、最大步数、简单停滞检测）。
    • 工具系统：统一的工具接口与简单管理器，支持安全等级与动态发现。

    详细组件分析¶

    Cron 工具示例¶

    • 功能概览
    • 支持三种操作：validate（校验）、parse（解析）、next（计算下一次执行时间）。
    • 输入参数：operation、expression、from（可选）、count（可选）。
    • 输出：校验返回布尔与字段集合；解析返回各字段字符串；next 返回多个 RFC3339 时间串。
    • 实现要点
    • 表达式解析：按 5 个字段拆分，支持通配符、枚举、范围、步长等语法。
    • 匹配算法：将时间逐分钟推进，检查是否满足所有字段集合。
    • 错误处理：参数缺失、格式非法、越界、超时搜索等均给出明确错误。
    • 使用方式
    • 在示例中直接创建工具实例并调用 Execute。
    • 在代理中注册到工具管理器，即可通过自然语言触发。
        flowchart TD
    Start(["开始"]) --> Op{"操作类型?"}
    Op --> |validate| V["解析表达式并校验"]
    Op --> |parse| P["解析表达式为字段"]
    Op --> |next| N["从指定时间起计算下 N 次"]
    V --> RetV["返回 {valid, fields}"]
    P --> RetP["返回 {expression, minute, hour, day, month, weekday}"]
    N --> Find["逐分钟推进匹配"]
    Find --> Done{"找到足够数量?"}
    Done --> |是| RetN["返回时间数组"]
    Done --> |否| Err["返回错误: 未在一年内找到足够次数"]
    RetV --> End(["结束"])
    RetP --> End
    RetN --> End
    Err --> End

    依赖分析¶

    • 组件耦合
    • Reactor 通过函数式选项注入 Thinker/Actor/Observer/Terminator，保持高内聚低耦合。
    • Thinker 依赖工具管理器与提示词构建器；Actor 依赖工具管理器；Observer 依赖上下文总线；Terminator 依赖上下文状态。
    • 外部依赖
    • Ollama 与 DashScope 客户端作为 LLM 适配层，Thinker 通过统一接口调用。
    • 循环依赖
    • 未见直接循环依赖；上下文在各组件间单向传递。

    性能考虑¶

    • 上下文压缩
    • 使用滑动窗口压缩策略与 Token 计数器，降低提示词长度，提升吞吐。
    • 流式输出
    • Thinker 通过流式接口实时输出思考片段，改善交互体验。
    • 工具执行计时
    • Actor 与 Thinker 分别记录工具与推理耗时，便于定位瓶颈。
    • 超时策略
    • 本地模型设置较长上下文超时；云端模型设置合理客户端超时，避免阻塞。
    • 最大步数限制
    • Terminator 限制最大迭代步数，防止长时间循环。

    故障排除指南¶

    • Cron 工具
    • 表达式格式错误：检查字段数量与特殊字符使用。
    • 越界值：分钟/小时/日期/月份/星期需在合法范围内。
    • 未找到足够次数：next 搜索限制为一年，超出会报错。
    • 代理示例
    • Ollama 连接失败：确认 BaseURL 与模型名正确，服务可达。
    • DashScope 缺少 API Key：确保环境变量已设置。
    • 本地模型冷启动慢：适当增加上下文超时。
    • 输出格式异常：Parser 无法解析 Action/ActionInput 时，会自动插入反思观察，提示重新组织输出。
    • 终止条件
    • 达到最大步数：调整 MaxSteps 或优化提示词与工具链。
    • 循环调用同一工具：检查工具参数与返回值是否导致无进展。

    结论¶

    本指南系统梳理了 GoReAct 的高级示例实现，重点在于： - Cron 工具的表达式解析与调度能力，可直接在代理中作为自然语言工具使用。 - Ollama 与 Qwen 两种部署形态的配置差异与执行流程一致性。 - 通过 Reactor、Thinker、Actor、Observer、Terminator 的协作，实现稳定可靠的 ReAct 循环。 - 结合上下文压缩、流式输出、计时与超时策略，达到性能与可用性的平衡。
