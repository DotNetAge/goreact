    基础示例¶

    简介¶

    本章节面向初学者与进阶开发者，系统讲解 GoReAct 基础示例的实现与使用，重点覆盖： - 本地 Ollama 模型集成与云端 DashScope（阿里云）模型集成的完整流程 - 测试用例与示例程序中的配置参数、客户端设置、Reactor 引擎初始化 - 工具管理器、思考器（Thinker）、执行器（Actor）的装配方式 - 环境变量配置、超时设置与错误处理最佳实践 - 运行命令、预期输出与常见问题解决方案

    项目结构¶

    该仓库采用按功能域分层的组织方式，基础示例位于 examples 目录，核心引擎与组件位于 pkg 目录。示例展示了两种运行模式： - 本地模式：通过 Ollama 客户端连接本地模型 - 云端模式：通过 OpenAI 兼容客户端连接 DashScope 模型

    核心组件¶

    • Reactor 引擎：组装 Thinker、Actor、Observer、Terminator 四大组件，驱动 ReAct 循环
    • Thinker：负责意图解析、工具发现、提示词合成、调用 LLM、解析输出
    • Actor：根据 Thinker 的动作选择具体工具并执行
    • Observer：将 Actor 的执行结果或错误序列化为文本反馈给 Thinker
    • Terminator：终止条件判断（完成、步数上限、循环检测）
    • 工具管理器与内置工具：提供计算器与日期时间工具

    下图展示从用户输入到最终结果的 ReAct 循环在引擎中的执行路径。

        sequenceDiagram
    participant U as "用户"
    participant R as "Reactor"
    participant T as "Thinker"
    participant A as "Actor"
    participant O as "Observer"
    participant M as "工具管理器"
    U->>R : "输入任务"
    R->>T : "推理与规划"
    T->>M : "列出可用工具"
    T->>T : "构建提示词并调用 LLM"
    T-->>R : "返回思考/动作/观察片段"
    R->>A : "执行动作"
    A->>M : "查找并执行工具"
    A-->>O : "返回原始结果/错误"
    O-->>R : "文本化观察结果"
    R->>R : "检查终止条件"
    alt 未完成
    R->>T : "进入下一轮循环"
    else 已完成
    R-->>U : "输出最终答案与统计"
    end

    本地 Ollama 示例（examples/ollama_agent/main.go）¶

    • 客户端设置
    • 使用 Ollama 客户端，BaseURL 指向本地服务，模型名与超时配置
    • 工具管理器
    • 注册计算器与日期时间工具
    • Reactor 初始化
    • 通过 WithThinker/WithActor/WithObserver/WithTerminator 组装引擎
    • 上下文与超时
    • 使用较长超时以适配本地模型冷启动与推理延迟
    • 实时思考流
    • 通过上下文选项注入思考流回调，边生成边打印

    运行要点 - 确保本地 Ollama 服务已启动且可访问 - 如需更长推理时间，可适当提高超时 - 观察思考流有助于调试与可视化推理过程

    故障排查指南¶

    • 环境变量未设置
    • 云端示例需设置 DASHSCOPEAPIKEY；否则测试会跳过或示例直接退出
    • 客户端初始化失败
    • 检查 BaseURL、API Key、模型名与超时配置
    • 推理无输出或结果为空
    • 检查思考流回调是否正确注入
    • 核对工具是否正确注册与可用
    • 超时导致提前终止
    • 适当增加上下文超时；确认网络与模型服务状态
    • 终止条件触发
    • 检查最大步数设置与是否存在循环调用同一工具的情况

    结论¶

    通过本基础示例，你可以快速掌握： - 如何在本地与云端两种环境下集成 LLM - 如何装配 Reactor 引擎与四大核心组件 - 如何配置工具管理器与内置工具 - 如何设置超时、启用思考流与进行错误处理 - 如何在测试与示例中验证执行结果与统计信息

    这些实践既适合入门学习，也为生产级集成提供了清晰的参考路径。
