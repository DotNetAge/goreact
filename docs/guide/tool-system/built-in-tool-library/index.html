    内置工具库¶

    简介¶

    本文件系统性梳理 GoReAct 内置工具库，覆盖文件操作、搜索浏览与执行扩展三类工具，帮助用户理解各工具的功能边界、参数规范、安全级别、错误处理与性能特征，并提供常见使用场景与最佳实践。新增 Cron 表达式解析工具，提供表达式验证、解析和执行时间计算功能。

    项目结构¶

    内置工具集中于 pkg/tools/builtin 目录，采用"按功能分组"的组织方式，每个工具一个独立文件，统一实现工具接口与通用校验逻辑，便于组合与扩展。

    核心组件¶

    • 工具接口与安全等级
    • 所有工具均实现统一的工具接口，包含名称、描述、执行与安全等级声明。安全等级用于指导运行时策略与权限控制。
    • 通用校验与安全
    • 提供必需参数校验、字符串参数校验与文件安全检查（如禁止访问敏感系统文件），保障工具调用的健壮性与安全性。
    • 工具分类
    • Tier 1：文件操作（Read、Write、Edit）
    • Tier 2：搜索浏览（Glob、Grep、LS）
    • Tier 3：执行扩展（Bash、Calculator、DateTime、Cron）

    架构总览¶

    内置工具遵循"单一职责、组合强大"的设计原则，通过统一的工具接口与参数模型协作，形成可复用的工具集。

    详细组件分析¶

    系统工具¶

    Bash（命令执行）¶

    • 功能概述
    • 在受控环境中执行 Bash 命令，返回输出、成功状态与错误信息。
    • 参数
    • command: string（必需）- 待执行的 Bash 命令。
    • 安全级别
    • 高风险（LevelHighRisk）。建议仅在可信环境与受控策略下启用。
    • 错误处理
    • 缺失或类型不匹配的 command 参数会直接报错；命令执行失败时返回错误详情。
    • 性能与限制
    • 受命令本身与系统资源影响；建议避免长时间阻塞命令。
    • 使用场景
    • 快速系统探测、脚本片段执行、一次性任务自动化。
    • 最佳实践
    • 严格限制可执行命令集合；对输入进行白名单校验；避免敏感系统命令；结合超时控制。

    依赖分析¶

    • 内聚性
    • 各工具职责单一，内部逻辑清晰，便于维护与测试。
    • 耦合度
    • 工具间无直接耦合，通过统一接口与参数模型交互。
    • 外部依赖
    • Bash 依赖系统 shell；Email 依赖网络协议栈；其余工具主要依赖标准库。
    • 安全钩子
    • 通过通用安全检查函数统一拦截高风险路径与敏感文件。

    性能考虑¶

    • IO 密集型工具
    • Read/Grep/LS/Write/Edit/Replace 等工具以文件系统与磁盘 IO 为主，建议：
      • 控制文件大小与数量；使用 include/recursive 精准定位。
      • 对超大文件设置上限（如 Read 的 10MB、Grep 的 5MB）。
    • 网络密集型工具
    • Email 依赖网络连接，建议：
      • 合理设置超时与重试；使用 TLS；避免高并发导致连接池耗尽。
    • CPU 密集型工具
    • Calculator/DateTime/Cron 为纯计算，性能开销极低；Cron 的搜索过程按分钟推进，建议限制 count 与 from。
    • 安全检查成本
    • validateFileSafety 与路径清理开销很小，但能显著降低风险。

    故障排除指南¶

    • 参数缺失或类型错误
    • 现象：直接返回"缺少必需参数"或"参数类型不匹配"。
    • 排查：核对必填字段与类型；使用 Echo 验证参数传递。
    • 文件相关错误
    • 现象：文件不存在、路径为目录、文件过大、写入失败。
    • 排查：确认路径与权限；缩小文件范围；检查磁盘空间。
    • 搜索与匹配错误
    • 现象：Glob/Grep 模式无效、未找到匹配。
    • 排查：修正通配符/正则；缩小搜索范围；确认 include 过滤。
    • 网络与认证错误（Email）
    • 现象：连接被拒、超时、认证失败。
    • 排查：检查服务器地址/端口/TLS；使用应用密码；确认防火墙与 2FA 设置。
    • Cron 表达式错误
    • 现象：字段数量不符、值越界、步长非法。
    • 排查：使用 validate 预检；对照标准格式；简化表达式。

    结论¶

    GoReAct 内置工具库以"单一职责、安全可控、易于组合"为核心设计思想，覆盖文件操作、搜索与执行扩展三大场景。新增的 Cron 工具进一步增强了时间调度和表达式解析能力，通过统一接口与通用校验机制，既保证了易用性，也提供了足够的安全边界。建议在生产环境中结合安全策略与监控体系，谨慎启用高风险工具，并对关键路径与数据进行备份与审计。

    附录¶

    工具安全级别一览¶

    • 安全（LevelSafe）：Calculator、DateTime、Echo、Glob、Grep、LS、Read、Cron。
    • 敏感（LevelSensitive）：Edit、Replace、Write。
    • 高风险（LevelHighRisk）：Bash、Email。
