    MCP 提供商集成¶

    简介¶

    本文件面向希望在 GoReAct 中集成 MCP（Model Context Protocol）提供商的开发者，系统阐述 MCP 协议在工具系统中的工作原理与应用方式，给出 Provider 接口的实现要求与集成步骤，提供 MCP 客户端的配置与连接方法（含认证、超时与健康检查），并说明如何将外部服务或本地工具通过 MCP 协议暴露为 GoReAct 可用的工具。文档同时包含注册、发现与调用流程的可视化图示，并提供常见问题的解决方案与性能优化建议。

    项目结构¶

    围绕 MCP 集成的关键代码位于以下模块： - 提供者接口与注册表：pkg/tools/provider - MCP 提供者实现：pkg/tools/provider/mcp - 工具抽象与安全等级：pkg/tools - 内置工具示例：pkg/tools/builtin - 示例与用法：examples

    核心组件¶

    • Provider 接口：定义提供者的统一能力模型（名称、初始化、发现工具、获取工具、关闭、健康检查）。
    • Registry 注册表：集中管理多个 Provider，支持注册、查询、批量发现工具与关闭。
    • MCPProvider：实现 Provider 接口，对接外部 MCP 服务器，负责健康检查、工具发现与远程工具执行。
    • MCPTool：对远端工具进行封装，作为 GoReAct 的 Tool 使用。
    • 工具接口与安全等级：定义工具的统一行为与安全风险分级，MCP 工具默认视为高风险。

    详细组件分析¶

    Provider 接口与注册表¶

    • Provider 接口定义了提供者的核心职责：命名、初始化（含认证与连接）、发现工具、按名获取工具、关闭与健康检查。
    • Registry 提供注册、查询、列出、批量发现工具与统一关闭的能力，是多提供者场景下的中枢。

    依赖关系分析¶

    • MCPProvider 依赖工具接口 Tool 以适配 GoReAct 的工具管理器。
    • MCPTool 依赖 MCPProvider 的 HTTP 客户端与服务器地址。
    • Registry 统一管理 Provider，支持批量发现工具与健康状态检查。

    性能考量¶

    • 超时设置：初始化时可配置超时秒数，避免阻塞；建议根据 MCP 服务器延迟与网络状况调整。
    • 健康检查：定期调用 /health 以检测可用性，异常时标记不健康并跳过该提供者。
    • 连接复用：HTTP 客户端默认复用连接，减少握手开销；关闭时清理空闲连接。
    • 工具缓存：发现工具后缓存为 MCPTool，避免重复网络请求。
    • 错误传播：网络错误与非 200 响应会触发不健康状态，有助于快速失败与降级。

    故障排查指南¶

    • 无法连接 MCP 服务器
    • 检查 server_url 是否正确，/health 是否可达。
    • 如启用认证，确认 API Key 设置正确。
    • 查看超时配置是否过短。
    • /tools 返回非 200 或解析失败
    • 确认服务器返回 JSON 结构包含 tools 数组，且每项包含 name、description、schema。
    • 观察健康状态变化（不健康时将跳过该提供者）。
    • /execute 执行失败
    • 检查请求体是否包含 tool 与 params。
    • 关注响应中的 success 与 error 字段，定位远端错误。
    • 安全与授权
    • MCP 工具默认高风险，需配合授权策略与 HITL（人工在回路）控制。

    结论¶

    通过 Provider 接口与 Registry，GoReAct 能够以统一方式集成 MCP 提供商。MCPProvider 将外部工具服务器抽象为本地 Tool，借助健康检查、超时与错误处理保障稳定性。结合安全等级与授权策略，MCP 工具可在保证可控的前提下扩展 Agent 的能力边界。建议在生产环境中为 MCP 服务器配置合理的超时、重试与熔断策略，并对高风险工具实施严格的授权控制。

    附录¶

    MCP 提供者集成步骤¶

    • 创建 MCP 提供者实例并初始化配置（serverurl、apikey、timeout）。
    • 调用 Initialize 完成健康检查与连接建立。
    • 调用 DiscoverTools 获取工具清单并注册到工具管理器。
    • 通过工具管理器按名称获取并执行工具。
    • 在生命周期结束时调用 Close 释放资源。
