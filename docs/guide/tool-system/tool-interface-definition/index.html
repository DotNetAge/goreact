    工具接口定义¶

    简介¶

    本文件面向 GoReAct 工具接口系统，系统性阐述 Tool 接口与 Manager 接口的设计理念、方法职责、安全等级划分与授权流程，并结合内置工具示例与安全钩子机制，给出实现最佳实践、常见问题与性能优化建议。读者无需深入底层即可理解如何扩展与安全地使用工具。

    项目结构¶

    GoReAct 的工具体系位于 pkg/tools 下，核心接口与默认管理器定义于该包；内置工具位于 pkg/tools/builtin，覆盖文件操作、搜索浏览与执行扩展三类场景；安全授权由 actor 包的 SecurityHook 实现，依据工具安全等级触发人机交互（HITL）。

    核心组件¶

    • Tool 接口：定义工具的唯一标识、能力描述、安全等级与执行入口，统一了工具的调用协议。
    • Manager 接口：负责工具发现与检索，支持简单注册表与语义检索等扩展。
    • MapTool：轻量封装，将任意函数包装为 Tool，便于快速实现自定义工具。
    • ExtractInput：将输入 map 严格反序列化为目标结构体，避免类型不一致导致的错误。
    • SimpleManager：内存注册表，默认返回全部工具或按意图动态筛选（可扩展）。

    架构总览¶

    工具接口贯穿“思考-决策-执行”链路：Thinker 通过 ToolManager 获取可用工具，Actor 调用具体 Tool.Execute 执行动作；SecurityHook 基于工具安全等级拦截高风险操作，必要时请求人工授权。

    详细组件分析¶

    Tool 接口与安全等级¶

    • 设计理念
    • 唯一性：Name() 必须与思考器调用名一致，保证精确路由。
    • 可解释性：Description() 用于系统提示词，帮助模型正确使用工具。
    • 安全性：SecurityLevel() 为授权与拦截提供依据。
    • 可执行性：Execute() 接收上下文与输入映射，返回结果或错误。
    • 安全等级
    • LevelSafe：纯查询、无副作用（如 Read、Calculator）。
    • LevelSensitive：有限写入或受限操作（如 Write）。
    • LevelHighRisk：任意执行、高风险（如 Bash、SQL Drop）。
    • 授权流程
    • SecurityHook 在 actor 步骤开始时拦截，若工具安全等级达到高风险且未白名单，则触发 HITL 授权；批准后自动会话内白名单放行。

    依赖分析¶

    性能考虑¶

    • 输入解析
    • 使用 ExtractInput 将 map 反序列化到结构体，减少后续类型断言与分支判断，降低运行时开销。
    • 文件操作
    • Read 对大文件设置上限，Write 自动创建目录并按需追加，避免频繁系统调用。
    • 工具检索
    • SimpleManager 采用内存 map，GetTool 与 ListAvailableTools 时间复杂度分别为 O(1) 与 O(n)，适合中小规模工具集；大规模场景建议引入向量化检索或缓存。
    • Cron 计算
    • next 计算带有最大迭代限制，防止表达式异常导致无限循环，保障稳定性。

    故障排查指南¶

    • 工具未找到
    • 检查工具名称是否与 Name() 完全一致；确认已通过 Manager.Register 注册。
    • 执行报错“缺少执行逻辑”
    • MapTool 的 ExecuteFunc 为空，需在构造时注入执行函数。
    • 参数缺失或类型错误
    • 使用 validateRequired/validateRequiredString 与 ExtractInput 统一校验与反序列化。
    • 高风险工具被阻断
    • 若未配置审批器或审批拒绝，将返回明确错误；可在会话中配置审批器或在长期记忆中建立白名单。
    • 文件访问受限
    • 特殊文件名保护与路径清理可能导致访问失败，检查路径与权限。

    结论¶

    GoReAct 的工具接口以简洁统一的契约实现了“可发现、可授权、可执行”的闭环：Tool 接口明确了工具的职责边界，Manager 提供灵活的发现机制，MapTool 与 ExtractInput 降低了实现成本，SecurityHook 则在关键节点保障安全。结合内置工具的最佳实践，开发者可以快速、安全地扩展自定义工具。
