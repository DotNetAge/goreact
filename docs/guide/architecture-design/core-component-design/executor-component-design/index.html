    执行器组件设计¶

    简介¶

    本文件面向执行器（Actor）组件，系统性阐述其在 ReAct 循环中的职责边界、工具调用与执行流程、安全拦截与授权机制、以及可观测性与性能监控。Actor 作为“双手”，负责将 Thinker 的结构化动作（Action）映射到具体工具并执行，同时承担安全拦截、错误恢复与指标采集等关键职责。本文以代码为依据，结合测试与示例，给出可操作的设计原则与最佳实践。

    项目结构¶

    Actor 组件位于 pkg/actor 目录，核心文件包括： - 接口定义：actor.go - 默认实现：defaultactor.go - 安全钩子：securityhook.go - 单元测试：defaultactortest.go、securityhooktest.go - 工具体系：tools/tool.go 及内置工具 calculator.go、bash.go、read.go - 上下文与类型：core/types.go、core/context.go - 指标系统：metrics/metrics.go - 引擎装配：engine/reactor.go - 示例：examples/agent_test.go

    核心组件¶

    • Actor 接口：定义 Act(ctx) 执行入口，负责从上下文提取最后一步动作、路由工具、执行并回填原始结果/错误。
    • 默认实现 defaultActor：基于工具管理器进行工具查找与执行；记录工具级延迟指标；将 rawoutput/rawerror 写入上下文。
    • 安全钩子 SecurityHook：在 Actor 步骤前拦截高风险工具，支持会话白名单与人工授权；授权失败时设置特定错误。
    • 工具体系：工具接口定义安全等级；SimpleManager 提供内存注册表；内置工具覆盖查询、编辑、执行等场景。
    • 上下文 PipelineContext：承载会话、轨迹、指标、附件、状态存储等；提供 LastTrace、Get/Set 辅助。
    • 指标系统：统一记录延迟、错误、Token 使用、资源占用等；默认实现支持并发安全聚合。
    • 引擎 Reactor：装配 Thinker、Actor、Observer、Terminator 与 Hook，驱动 ReAct 循环。

    Actor 在 ReAct 循环中的位置与职责如下：

    详细组件分析¶

    Actor 接口与职责¶

    • 职责边界：仅负责动作执行与结果回填，不参与决策与观察；内部处理重试与超时边界。
    • 输入输出：从 PipelineContext.LastTrace().Action 读取动作；将原始输出/错误写入上下文键值。
    • 并发与超时：建议在工具执行侧使用 context 控制超时与取消；Actor 层负责工具级重试与熔断策略。

    依赖分析¶

    • Actor 依赖工具管理器与上下文；通过指标系统记录延迟。
    • SecurityHook 依赖工具管理器、记忆银行与 Approver 回调；拦截 Actor 步骤。
    • 引擎装配将 Actor 与 Hook 注入流水线，驱动 ReAct 循环。

    性能考虑¶

    • 工具级延迟：Actor 按工具名记录延迟，便于定位慢工具与热点路径。
    • 并发与超时：建议在工具 Execute 中使用 context 控制超时；Actor 层可实现指数退避重试（当前默认实现聚焦工具路由与指标）。
    • 资源监控：通过指标系统记录 CPU/内存/GPU 使用，结合工具延迟分析资源瓶颈。
    • 日志与可观测：上下文日志与指标双通道，便于问题复盘与容量规划。

    故障排查指南¶

    • 动作为空：Act 返回空结果；检查 Thinker 是否正确生成动作。
    • 工具未注册：记录警告并写入空结果；确认工具管理器已注册对应工具。
    • 工具执行失败：raw_error 写入上下文；检查工具输入参数与权限。
    • 安全拦截：
    • 未配置 Approver：报错提示需要人工授权；补充 approver 回调。
    • 用户拒绝：设置特定错误；确认授权流程与白名单策略。
    • 指标缺失：确认指标系统已注入上下文；检查工具名与标签键一致。

    结论¶

    Actor 作为 ReAct 循环的执行核心，承担动作路由、工具执行、安全拦截与可观测性职责。通过清晰的接口设计、可插拔的工具管理器与安全钩子，Actor 在保证安全性的同时提供了良好的扩展性与可观测性。配合引擎装配与指标系统，可实现稳定、可控、可审计的自动化执行。

    附录¶

    配置管理与最佳实践¶

    • 工具注册：集中管理工具注册，确保安全等级显式声明；对高危工具启用安全钩子。
    • 安全授权：在受控环境中挂载安全钩子；合理使用会话白名单降低交互成本。
    • 指标采集：为关键工具命名规范，便于指标聚合与告警；定期审查延迟与错误分布。
    • 日志与追踪：在上下文中保留必要的状态键，便于调试与审计。
