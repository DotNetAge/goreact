    架构概览¶

    引言¶

    本架构文档面向 GoReAct 框架，系统性阐述其 ReAct（推理+行动）模式的实现理念与工程化落地。文档聚焦以下目标： - 解释四核认知组件（思考者、行动者、观察者、终结者）的职责与协作机制 - 描述 Reactor 引擎作为中央协调器的角色与“思考-行动-观察-检查”的循环执行模型 - 阐明插件化架构、接口隔离与模块解耦策略 - 给出系统边界、组件职责与依赖关系图 - 说明架构决策的技术考量、性能优化与可扩展性设计

    项目结构¶

    仓库采用按能力域划分的模块化组织方式，核心围绕 engine、agent、thinker、actor、observer、terminator、steps、tools、skill、core 等包展开。下图给出高层结构示意。

    核心组件¶

    • 思考者（Thinker）：负责意图识别、工具/技能编排、检索增强（RAG）与基于历史的下一步决策，产出“思考+动作”决策。
    • 行动者（Actor）：负责将思考者决策映射为具体工具调用或技能执行，处理并发、重试与超时，并回传原始结果或错误。
    • 观察者（Observer）：将行动者的原始输出/错误转化为结构化、降噪后的认知反馈，供思考者在下一轮迭代中使用。
    • 终结者（Terminator）：安全边界与终止条件判断，防止无限/停滞循环，成功/失败阈值、最大步数、资源耗尽等场景下优雅收尾。
    • Reactor 引擎：装配四核组件，构建可插拔逻辑管线，驱动 ReAct 循环；提供钩子扩展点与可观测性注入。
    • Agent：面向应用的富领域模型封装，暴露 Chat/Execute 等高层 API，内嵌技能记录与记忆体系。

    详细组件分析¶

    Reactor 引擎（中央协调器）¶

    • 职责
    • 组装四核组件（Thinker/Actor/Observer/Terminator），校验依赖完整性
    • 构建共享执行上下文（PipelineContext），注入日志与指标
    • 将步骤封装为管道步骤并注册钩子，驱动循环执行
    • 在每次循环间进行短间隔休眠以避免忙等
    • 关键点
    • 通过函数式选项注入组件与钩子，支持插件化替换
    • 通过管道 Hook 扩展（如安全授权拦截）而不侵入核心流程
    • 通过上下文注入全局可观测性，便于端到端追踪

    依赖分析¶

    • 组件耦合与内聚
    • Reactor 仅依赖接口（Thinker/Actor/Observer/Terminator），通过函数式选项注入，高内聚低耦合
    • Steps 仅依赖接口与 Pipeline，保持对核心引擎的最小依赖
    • Agent 作为上层封装，依赖 Reactor 与技能/记忆，但不直接耦合具体实现
    • 外部依赖
    • 通用逻辑管线（gochat/pkg/pipeline）用于步骤编排与钩子
    • 日志与指标通过 core.Logger/Metrics 注入，便于替换实现
    • 潜在风险
    • 若未注入四核组件，Reactor 将在运行期报错，需在装配阶段显式校验
    • Actor 步骤在执行前会检查上下文中的错误（例如安全拦截），应确保钩子正确设置

    性能考虑¶

    • 循环节流
    • 每次循环后短暂休眠，避免 CPU 忙等，适合外部事件驱动或长耗时工具场景
    • 上下文与令牌统计
    • PipelineContext 统计令牌用量与开始时间，便于成本控制与性能分析
    • 工具与技能
    • 工具管理器可按意图筛选，减少无关工具参与推理，降低上下文膨胀
    • 技能统计与评分驱动“指令集”持续优化，间接提升整体吞吐与成功率
    • 可观测性
    • 通过 WithThoughtStream 实现链上思考流的实时反馈，利于前端体验与调试

    故障排查指南¶

    • 常见问题
    • 组件缺失：若未注入 Thinker/Actor/Observer/Terminator，运行时报错提示必须配置四核组件
    • 安全拦截：Actor 步骤在执行前检查上下文错误，若存在授权拒绝或前置钩子注入的错误，将中断当前循环
    • 终止条件：Terminator 返回停止信号时，引擎会锁定状态并标记完成；检查 FinishReason 与错误信息定位原因
    • 建议排查步骤
    • 启用日志与指标，确认每步的思考、行动、观察与终止判断
    • 使用 WithThoughtStream 观察链上思考流，定位推理卡点
    • 检查工具安全等级与权限策略，确保敏感操作符合授权要求
    • 校验最大步数与资源限制，避免误判为死循环

    结论¶

    GoReAct 以“思考-行动-观察-检查”的 ReAct 循环为核心，通过 Reactor 引擎将四核认知组件以管道步骤形式编排，实现高度可插拔、可扩展与可演化的智能体执行框架。接口隔离与模块解耦使系统具备良好的可维护性与可测试性；工具与技能体系进一步增强了任务分解与自适应能力。结合钩子与可观测性，可在生产环境中实现安全可控、性能可度量的智能体执行流水线。

    附录¶

    • 架构决策要点
    • 采用函数式选项注入组件，简化装配复杂度，便于替换与测试
    • 将“思考-行动-观察-检查”固化为标准步骤，保证执行一致性
    • 通过 Pipeline Hook 支持安全拦截与审计，不侵入核心逻辑
    • 可扩展性设计
    • 新增工具：实现 Tool 接口并注册到管理器
    • 新增技能：遵循技能规范，利用统计与评分驱动持续优化
    • 新增终结规则：实现 Terminator 接口，按业务需求扩展终止条件
