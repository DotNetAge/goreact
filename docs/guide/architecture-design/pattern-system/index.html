    模式系统¶

    简介¶

    GoReAct 是一个高性能、可编程的 ReAct（推理 + 行动）框架，专注于智能体任务编排和模式系统。该框架引入了创新的"三态转化"架构，将技能从"源码态"进化到"编译态"再到"执行态"，实现了从"提示词工程"到"智能体编译"的跨越。

    框架的核心特色包括： - 三模态记忆系统：工作记忆（短期）、语义记忆（长期/RAG）、肌肉记忆（经验） - 通用逻辑管道：原生支持条件判断、循环、中断、返回等逻辑 - 思维暗语驱动模式：/plan、/specs 等 codeword 驱动的思考模式 - 自适应快径执行：基于预期修正的智能体编译和执行

    核心组件¶

    编译态数据结构¶

    编译态是 Evo 模式的核心，它将成功的 ReAct Traces 蒸馏为确定性的执行图。

        classDiagram
    class CompiledAction {
    +string SkillName
    +JSONSchema InputSchema
    +[]ActionStep Steps
    +string SuccessCriteria
    }
    class ActionStep {
    +string ID
    +string ToolName
    +string InputTemplate
    +string ExpectedObservation
    +string ValidationRule
    +string Description
    }
    CompiledAction --> ActionStep : "包含多个步骤"

    架构概览¶

    GoReAct 的模式系统实现了从"源码态"到"执行态"的完整进化链路：

        flowchart TD
    Start([用户意图]) --> Intent{意图识别}
    Intent --> |存在肌肉记忆| FastPath[加载编译态执行图]
    Intent --> |无记忆/新技能| SourceMode[源码态: 建立理想预期]
    subgraph "解释执行期 (SourceMode)"
    SourceMode --> MasterPlan[Master: 任务拆解]
    MasterPlan --> TaskType{原子 vs 复合任务?}
    TaskType --> |原子| AtomicTool[原子工具执行]
    TaskType --> |复合| SubAgent[子代理 ReAct 循环]
    AtomicTool --> CollectObs[收集观察结果]
    SubAgent --> CollectObs
    CollectObs --> Success{执行成功?}
    Success --> |是| Compile[编译器: 提取执行图]
    Success --> |否| Error[错误处理]
    Compile --> SaveMemory[保存到肌肉记忆]
    SaveMemory --> FastPath
    end
    subgraph "自适应快径执行 (FastPath)"
    FastPath --> ExecStep[执行原子步骤]
    ExecStep --> Compare{实际 vs 预期?}
    Compare --> |匹配| NextStep{还有步骤?}
    Compare --> |不匹配| Escalate[升级: 唤醒全量 ReAct]
    NextStep --> |是| ExecStep
    NextStep --> |否| FinalAnswer[完成任务]
    Escalate --> Detective[侦探 Thinker 分析误差原因]
    Detective --> UpdateExpect[修正预期指纹]
    UpdateExpect --> SaveMemory
    end

    详细组件分析¶

    编译器组件¶

    编译器负责将成功的 ReAct Traces 转换为编译态执行图：

        sequenceDiagram
    participant User as 用户
    participant Compiler as 编译器
    participant Thinker as 思考者
    participant Memory as 记忆系统
    User->>Compiler : Compile(skillName, traces)
    Compiler->>Compiler : 序列化 traces 为 JSON
    Compiler->>Compiler : 构造编译提示词
    Compiler->>Thinker : Think(pipelineContext)
    Thinker-->>Compiler : 返回编译后的执行图
    Compiler->>Compiler : 设置 skillName
    Compiler-->>User : 返回 CompiledAction
    Note over Compiler,Thinker : 使用 /json 协议直通 LLM

    性能考虑¶

    内存优化策略¶

    1. 肌肉记忆缓存：使用内存存储 CompiledAction，支持快速查找和更新
    2. 状态变量管理：仅在需要时存储中间状态，避免内存泄漏
    3. 连接池复用：在编译器和思考者之间共享资源

    执行效率优化¶

    1. 快径优先：优先尝试编译态执行，减少 LLM 调用次数
    2. 增量编译：只在执行成功后进行编译，避免无效工作
    3. 并行执行：在支持的情况下并行执行独立任务

    错误恢复机制¶

    1. 渐进式降级：从编译态到源码态的平滑过渡
    2. 状态回滚：在升级过程中保持执行状态的一致性
    3. 重试策略：对偶发性错误提供自动重试机制

    故障排除指南¶

    常见问题诊断¶

    1. 编译失败：检查 /json 协议是否正确设置，验证 LLM 输出格式
    2. 预期不匹配：确认 ExpectedObservation 是否过于严格，考虑添加 ValidationRule
    3. 升级循环：监控升级频率，避免过度依赖全量 ReAct

    调试技巧¶

    1. 日志分析：利用 Logger 记录详细的执行轨迹
    2. 状态检查：定期检查变量状态和内存使用情况
    3. 性能监控：监控编译器和执行器的性能指标

    结论¶

    GoReAct 的模式系统通过创新的"三态转化"架构，实现了从"提示词工程"到"智能体编译"的技术跨越。该系统的核心价值体现在：

    1. 智能化进化：通过肌肉记忆实现技能的持续优化和自我修复
    2. 高效执行：编译态执行大幅减少了 LLM 调用和推理成本
    3. 稳健性保障：主从编排模式提供了可靠的降级策略和错误恢复机制
    4. 可扩展性：模块化的架构设计便于功能扩展和定制

    该模式系统为构建高性能、可编程的智能体应用提供了坚实的基础，特别适合需要复杂任务编排和持续学习的应用场景。
