    日志和指标¶

    简介¶

    本章节面向 GoReAct 的日志与指标体系，系统化阐述以下内容： - 日志系统：统一 Logger 接口设计、Zap 适配器实现、字段构造工具与使用建议。 - 指标系统：指标接口与默认实现、计时器/计数器/Token/资源使用统计模型、资源快照与增量计算。 - 集成关系：如何在 Reactor 与 PipelineContext 中注入日志与指标，以及在 Thinker 等组件中实际使用。 - 最佳实践：性能优化、准确性保障、告警配置思路与常见问题排查。

    项目结构¶

    日志与指标相关代码主要分布在如下模块： - 日志：pkg/log/logger.go 定义接口与字段工具；pkg/log/zaplogger.go 提供 Zap 适配器。 - 指标：pkg/metrics/metrics.go 定义指标接口与默认实现；pkg/metrics/resourcemonitor.go 提供运行时资源快照能力。 - 观测性抽象：pkg/core/observability.go 定义核心 Logger/Metrics 接口及空实现。 - 上下文与引擎：pkg/core/context.go 注入日志与指标；pkg/engine/reactor.go 将全局日志/指标注入到执行上下文；pkg/thinker/default_thinker.go 展示在具体组件中记录延迟与令牌用量的实际用法。

    核心组件¶

    • 日志接口与字段工具
    • Logger 接口定义了 Debug/Info/Warn/Error 与 With 子 Logger 创建方法，便于按上下文打点。
    • 字段工具函数支持字符串、整型、浮点、布尔、任意类型、错误与持续时间等常用字段类型，便于结构化输出。
    • Zap 适配器
    • 提供生产与开发两种配置，默认生产环境使用 ISO 时间戳与 Info 级别，开发环境使用开发配置。
    • 将内部字段转换为 zap.Field 并委托底层 zap.Logger 执行。
    • 指标接口与默认实现
    • 支持记录延迟、错误、成功、Token 消耗、系统资源使用；提供聚合查询与重置能力。
    • 默认实现包含计时器（含 min/max/avg）、计数器、Token 计数器、资源使用计数器，均采用互斥保护。
    • 资源监控
    • 提供运行时快照（内存、堆、栈、Goroutine、GC 等）与快照差分，便于计算区间内的资源变化。
    • 观测性抽象与上下文注入
    • core 包提供 Logger/Metrics 抽象与空实现，PipelineContext 默认注入空实现，Reactor 可注入全局实现。
    • 引擎集成
    • Reactor 在 Run 时将全局 Logger/Metrics 注入到 PipelineContext，组件可在执行过程中直接使用。

    架构总览¶

    日志与指标贯穿引擎执行链路：Reactor 将全局 Logger/Metrics 注入 PipelineContext，随后在 Thinker 等步骤中记录延迟、令牌用量与资源使用，最终由默认实现进行聚合与导出。

    详细组件分析¶

    日志系统¶

    • 接口与字段
    • Logger 接口统一了不同日志库的抽象，With 方法可派生带预设字段的子 Logger，便于跨模块复用。
    • 字段工具函数覆盖常见类型，Err 与 Duration 等便于标准化输出。
    • Zap 适配器
    • 生产与开发两种配置，时间键名与编码器可定制；With 派生新 Logger 时透传底层 zap.Logger.With。
    • convertFields 将内部字段映射为 zap.Field，确保与 zap 生态兼容。
    • 使用建议
    • 在 Reactor 层注入全局 Logger，组件通过 PipelineContext.Logger 使用，避免在各层重复初始化。
    • 对关键路径（如 Think/Act/Observe）记录 Info 级别事件，配合 With 添加会话、步骤、输入摘要等上下文字段。

    依赖分析¶

    • 松耦合设计
    • 日志与指标通过接口抽象，Reactor 仅依赖 core.Logger/core.Metrics，不关心具体实现。
    • PipelineContext 作为承载者，集中注入 Logger/Metrics，组件只需持有上下文即可使用。
    • 外部依赖
    • 日志：Zap 生态（zap/zapcore），用于高性能结构化日志。
    • 指标：标准库 sync/时间包，用于并发控制与时间度量。
    • 循环依赖
    • 未发现循环导入；日志与指标模块均保持自包含，通过 core 抽象与上下文进行连接。

    性能考虑¶

    • 日志
    • 使用 With 派生子 Logger，减少重复字段写入成本；避免在高频路径构造复杂字段。
    • 生产环境默认 Info 级别，必要时在开发环境开启 Debug 以降低生产开销。
    • 指标
    • DefaultMetrics 在 GetMetrics 时先复制引用再解锁，避免在持有锁期间调用嵌套方法，提高并发吞吐。
    • 计时器/计数器/Token/资源使用均采用互斥保护，尽量减少锁持有时间。
    • 资源监控
    • Snapshot 读取 runtime.MemStats 与运行时状态，建议在关键步骤前后采样，避免过于频繁导致额外开销。
    • 建议
    • 将高成本字段（如对象序列化）延迟到需要时再构造。
    • 对于高频事件，优先使用计数器而非频繁写入日志。

    故障排查指南¶

    • 日志无法输出
    • 检查是否正确注入 Logger 到 PipelineContext；确认使用 With 派生的子 Logger 是否传递到子模块。
    • 若使用默认空 Logger，需显式注入有效实现。
    • 指标为空或异常
    • 确认是否调用了 Record* 方法；检查 operation 名称是否一致。
    • 使用 Reset 后指标清零属正常行为；如需持久化聚合，应在外部保存 GetMetrics 结果。
    • 资源监控偏差
    • Snapshot 与 Delta 依赖时间间隔，确保两次采样之间有足够时间；避免在极短时间内多次采样。
    • 单元测试参考
    • 指标聚合与重置行为可通过测试验证：metrics_test.go:8-61

    结论¶

    GoReAct 的日志与指标体系通过清晰的接口抽象与默认实现，实现了高性能、可观测且易扩展的运行时观测能力。日志方面，统一接口与 Zap 适配器提供了灵活的结构化输出；指标方面，多维统计与并发安全设计满足生产级需求。通过 Reactor 与 PipelineContext 的集成，组件可在不感知实现细节的情况下完成日志与指标记录，从而获得一致的可观测体验。
