    核心引擎¶

    简介¶

    本章节面向 GoReAct 核心引擎（Reactor）的使用者与贡献者，系统性阐述引擎的初始化与配置、组件装配与执行流程，并深入解析“思考器-执行器-观察器-终止检查”四步循环的实现细节。文档同时提供可直接参考的代码片段路径、常见问题与优化建议，帮助初学者快速上手，也为资深开发者提供足够的技术深度。

    项目结构¶

    GoReAct 将 ReAct 循环抽象为一个可插拔的管道引擎，核心位于 pkg/engine，围绕四个认知组件（Thinker、Actor、Observer、Terminator）构建步骤化流水线。上下文状态由 pkg/core 统一管理，各组件通过接口解耦，便于替换与扩展。

    核心组件¶

    • Reactor：装配 Thinker/Actor/Observer/Terminator 四大组件，构建 ReAct 执行管线，驱动循环直至终止条件满足。
    • PipelineContext：承载会话标识、输入、当前步数、是否完成、最终结果、追踪记录、附件、最大步数、日志、指标、令牌用量、开始时间、错误、思考流回调等。
    • 四步管道：
    • 思考器步骤：基于历史与工具生成下一步 Thought/Action。
    • 执行器步骤：根据最后 Trace 的 Action 调用工具并写入中间态。
    • 观察器步骤：将执行结果清洗、序列化、降噪并附加到 Trace。
    • 终止检查步骤：判定成功、失败或边界超限，决定是否结束循环。

    架构总览¶

    Reactor 在 Run 中完成以下关键动作： - 注入全局日志与指标到上下文选项； - 构建 PipelineContext； - 校验四大组件是否齐全； - 组装四步管道并注册钩子； - 进入主循环：执行一次完整周期（Think→Act→Observe→Check），若未完成则短暂休眠后继续，直到终止。

    Reactor 初始化与配置¶

    • 选项模式（functional options）支持注入自定义日志、指标、管道钩子以及 Thinker/Actor/Observer/Terminator 实现。
    • 默认日志与指标来自 core 包；可通过 WithEngineLogger/WithEngineMetrics 注入。
    • WithPipelineHook 支持在管道执行期间插入自定义钩子。
    • NewReactor 接受任意数量 Option 并应用到内部实例。

    故障排查指南¶

    • 组件缺失：若任一核心组件未注入，Run 将返回初始化失败错误。请确保 WithThinker/WithActor/WithObserver/WithTerminator 均已配置。
    • 安全钩子拒绝：Actor 步骤会检查上下文中的错误字段，若存在则直接返回，导致当前循环提前终止。检查前置钩子逻辑与工具权限。
    • 工具未注册：默认 Actor 在找不到工具时会记录告警并写入空输出/错误，避免崩溃。请确认工具管理器已正确注册所需工具。
    • 输出格式异常：默认 Observer 会尝试将非字符串/字节输出序列化为 JSON。若序列化失败，会返回不可解析二进制的描述文本。请检查工具返回类型。
    • 终止条件：若长时间未完成，优先检查最大步数限制与终止器规则；同时关注停滞检测日志。

    结论¶

    Reactor 以清晰的接口与管道化设计，将 ReAct 的“思考-执行-观察-终止”闭环抽象为可插拔、可观测、可扩展的引擎。通过上下文统一状态、步骤封装职责、默认实现提供开箱即用能力，既适合快速集成，也便于深度定制。配合示例工程与测试用例，用户可以高效地搭建从简单到复杂的智能代理。
