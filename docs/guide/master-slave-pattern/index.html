    主从模式¶

    简介¶

    主从模式（Master-Slave Pattern）是 GoReAct 框架中实现复杂任务分解和执行的核心架构模式。该模式通过一个中央协调器（Master）负责高层次的任务规划和分解，以及多个子代理（Slave/Sub）负责具体任务的执行，实现了高效的分布式智能体协作。

    在 GoReAct 中，主从模式不仅支持单一层次的任务分解，还具备扩展能力，可以支持多级嵌套的主从关系，形成复杂的任务执行网络。这种设计使得系统能够处理从简单到复杂的各种任务场景，从单工具操作到多步骤的复合任务。

    项目结构¶

    GoReAct 项目采用模块化设计，主从模式位于 pkg/pattern/mastersub 目录下，与其他核心模块协同工作：

    核心组件¶

    Master（协调器）¶

    Master 是主从模式的中央控制器，负责高层次的任务规划和分解。它使用高级思维能力（Thinker）来分析用户目标，并将其转换为可执行的任务序列。

    关键特性： - 任务分解：将复杂目标拆分为原子任务 - 依赖管理：识别和管理任务间的依赖关系 - JSON 协议：使用标准化的 JSON 格式输出任务计划 - 重规划能力：支持基于失败或环境变化的任务重规划

    Sub-Reactor（子反应器）¶

    Sub-Reactor 是主从模式的执行单元，负责具体任务的实际执行。它基于 Reactor 引擎实现，提供完整的 ReAct 循环能力。

    核心功能： - 任务执行：执行具体的 Task 对象 - ReAct 循环：集成思考、行动、观察、检查的完整循环 - 状态跟踪：记录执行过程中的所有状态变化 - 结果封装：将执行结果封装为 TaskResult

    Orchestrator（编排器）¶

    Orchestrator 是主从模式的协调中心，负责管理 Master 和 Sub-Reactor 之间的交互，确保任务按正确的顺序和依赖关系执行。

    主要职责： - 初始化：设置 Master 和 Sub-Reactor 的协作关系 - 任务调度：管理任务的执行顺序和依赖检查 - 错误处理：处理执行过程中的各种异常情况 - 结果聚合：收集和整理所有任务的执行结果

    O-->>U : 返回最终结果

    性能考虑¶

    主从模式在设计时充分考虑了性能优化和资源管理：

    并发执行策略¶

    虽然当前实现采用串行执行以确保依赖关系正确，但系统架构支持并发执行的扩展：

        flowchart TD
    A[任务分解] --> B{任务数量}
    B --> |1| C[串行执行]
    B --> |>1| D[并行执行]
    D --> E[任务分组]
    E --> F[依赖图构建]
    F --> G[拓扑排序]
    G --> H[分层并发执行]
    C --> I[简化控制流]
    H --> I

    内存管理¶

    1. Trace 复制：在 Sub-Reactor 中进行值复制以确保解耦
    2. 上下文隔离：每个任务都有独立的执行上下文
    3. 资源清理：自动清理中间状态和临时数据

    缓存策略¶

    1. 任务结果缓存：已完成任务的结果可以被后续任务复用
    2. 依赖关系缓存：避免重复计算任务间的依赖关系
    3. 执行状态缓存：维护任务执行状态的快速查询

    常见问题诊断¶

    任务依赖失败¶

    症状：执行过程中出现依赖未满足的错误

    诊断步骤： 1. 检查任务 ID 的一致性 2. 验证依赖关系的正确性 3. 确认任务执行顺序

    解决方案： - 修正任务 ID 格式 - 重新设计任务依赖关系 - 实现更灵活的依赖检查机制

    LLM 输出格式错误¶

    症状：Master 分解失败，提示 JSON 解析错误

    诊断步骤： 1. 检查提示词模板的完整性 2. 验证 LLM 的输出格式 3. 确认 JSON 结构的有效性

    解决方案： - 优化提示词模板 - 实现输出格式验证 - 添加格式纠正机制

    执行超时问题¶

    症状：子任务执行时间过长

    诊断步骤： 1. 分析任务复杂度 2. 检查工具调用效率 3. 监控系统资源使用

    解决方案： - 实现任务超时机制 - 优化工具调用策略 - 添加执行进度监控

    结论¶

    主从模式作为 GoReAct 框架的核心架构模式，展现了现代智能体系统设计的最佳实践。通过明确的职责分离、强大的可扩展性和完善的错误处理机制，该模式为复杂任务的自动化执行提供了可靠的解决方案。

    主要优势¶

    1. 架构清晰：职责分离明确，易于理解和维护
    2. 扩展性强：支持多级嵌套和动态扩展
    3. 容错性好：单点故障不影响整体执行
    4. 可观测性：完整的执行追踪和状态记录

    发展方向¶

    1. 并发执行：实现更高效的并行任务执行
    2. 智能调度：基于任务特征的自适应调度算法
    3. 学习能力：集成机器学习算法优化任务分解
    4. 云原生支持：容器化和微服务化的部署方案

    主从模式不仅为当前的应用场景提供了有效的解决方案，也为未来的智能化发展奠定了坚实的基础。通过持续的优化和扩展，这一架构模式将在更广泛的领域发挥重要作用。
