    技能编译器¶

    引言¶

    本文件围绕 GoReAct 的“技能编译器”展开，系统阐述如何将临时的技能知识（肌肉记忆与短期会话记忆）转化为持久的记忆形式（SKILL.md），并在此基础上实现技能的演化、参数优化、变异与重组，以及质量控制与收敛性保障。文档同时给出配置与使用指引，并针对技能退化、收敛陷阱等常见问题提供解决方案。

    项目结构¶

    技能编译器位于 pkg/skill 子系统，配合记忆系统与 Agent 生命周期，形成从经验沉淀到技能固化再到新技能发现的闭环。

    核心组件¶

    • 知识编译器接口（Compiler）
    • RefineSkill：基于肌肉记忆对既有技能进行二次提炼与覆写。
    • DiscoverNewSkill：基于短期会话记忆发现长尾工作流，自动生成并注册新技能。
    • 技能数据结构（Skill）
    • 包含 YAML frontmatter（名称、描述、许可证、兼容性、元数据、允许工具）、正文指令、可选目录（scripts/references/assets）与运行时统计。
    • 技能管理器（Skill Manager）
    • 负责加载/注册/获取/列举技能；按任务选择最优技能；记录执行结果并维护统计数据；执行技能演化（优胜劣汰）与定时调度；归档/恢复技能。
    • 技能规范（Agent Skills 规范）
    • 明确 SKILL.md 的目录结构、frontmatter 字段约束、正文内容组织与可选资源目录。

    详细组件分析¶

    知识编译器接口（Compiler）¶

    • 职责
    • RefineSkill：扫描同一任务下的高权重肌肉记忆，利用大模型对历史经验进行二次总结蒸馏，覆写现有 SKILL.md。
    • DiscoverNewSkill：分析长期沉淀的短期会话记忆，寻找缺乏专属技能的长尾工作流，自动撰写全新 SKILL.md 并注册。
    • 设计要点
    • 与 MemoryBank 协作，从肌肉记忆与短期会话记忆中抽取高价值经验。
    • 通过 LLM 对经验进行归纳与压缩，确保指令清晰、可执行、可复用。

    依赖关系分析¶

    • 组件耦合
    • Compiler 依赖 MemoryBank（短期/经验记忆）与 LLM 客户端（语义选择与提炼）。
    • SkillManager 依赖 Compiler（Refine/Discover）、MemoryBank（经验数据）、LLM 客户端（语义选择）。
    • Agent/Builder 将 SkillManager 注入到执行链路，形成“执行—反馈—编译”的闭环。
    • 外部依赖
    • LLM 客户端（gochat core）用于语义匹配与经验提炼。
    • YAML 解析器用于 SKILL.md frontmatter 解析。

    性能考量¶

    • 选择模式权衡
    • KeywordOnly：快速但可能不够精确。
    • SemanticOnly：精确但较慢，需 LLM 客户端。
    • Hybrid：推荐，默认 TopN 候选，先关键词筛选，再语义精筛，失败时回退关键词最高分。
    • 评分与收敛
    • 综合评分采用加权聚合，避免单一指标主导。
    • 频率评分鼓励常用技能，效率评分限制过度优化导致的超参滥用。
    • 演化调度
    • 定时器周期扫描，避免实时重算带来的抖动。
    • 演化条件（长期未使用 + 低评分）防止资源浪费与技能退化。

    故障排查指南¶

    • 无合适技能
    • 现象：关键词筛选后无候选或语义匹配失败。
    • 处理：检查任务描述关键词覆盖度、技能描述关键词匹配、混合模式候选阈值与 LLM 客户端配置。
    • 语义选择异常
    • 现象：LLM 返回技能名无法匹配。
    • 处理：确认提示词构造、返回内容清洗（去引号/换行）、候选集合完整性。
    • 演化误杀
    • 现象：新技能或低频技能被归档。
    • 处理：调整演化条件（评分阈值、时间窗口）、启用人工干预与恢复流程。
    • 记录统计异常
    • 现象：成功率/效率/频率/综合评分异常。
    • 处理：核对 RecordExecution 输入参数（成功/耗时/Token/质量）、检查基准时间与总任务数更新逻辑。

    结论¶

    技能编译器通过“经验蒸馏 + 新技能发现”的双轮驱动，将 Agent 的执行经验固化为可复用的技能资产，并在管理器的评分与演化机制下持续优化。结合三模态记忆体系与闭环反馈，系统实现了从“新手指南”到“老司机”再到“创造者”的认知跃迁，具备良好的稳定性、可扩展性与可维护性。

    技能质量控制与收敛性保障¶

    • 质量控制
    • 质量评分总和与评估次数用于计算平均质量评分，避免极端样本影响。
    • 允许工具列表（allowed-tools）限制工具使用范围，降低风险。
    • 过拟合预防
    • 频率评分鼓励通用技能，抑制过度专用化。
    • 演化调度定期清理低效技能，防止局部最优陷阱。
    • 收敛性保证
    • 综合评分加权聚合，平衡成功率、效率、质量与频率。
    • 演化条件设置时间窗与评分阈值，避免长期停滞。
