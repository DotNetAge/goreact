    技能系统¶

    简介¶

    本文件系统性阐述 GoReAct 技能系统的设计与实现，覆盖以下主题： - 技能管理器的接口定义与默认实现，包括技能生命周期管理（加载、注册、选择、统计、排名、归档、恢复、演化与定时调度）。 - 技能规范文件格式（SKILL.md）与验证规则，以及技能描述与元数据的组织方式。 - 知识处理编译器（Compiler）的工作原理与"现有技能打磨""新技能发现"的进化机制，支持从记忆系统中提取高价值经验并反写为持久化的技能。 - 技能系统与记忆系统的集成关系，从短期记忆、经验记忆到技能沉淀的闭环。 - 技能开发最佳实践、性能优化与测试策略，以及常见冲突与版本管理建议。

    项目结构¶

    技能系统位于 pkg/skill 目录，围绕"技能定义、加载、选择、统计、演化"构建；记忆系统位于 pkg/memory，提供短期记忆与经验记忆的抽象；工具系统位于 pkg/tools，为技能执行提供工具能力。

    核心组件¶

    • 技能管理器接口与默认实现：负责技能的加载、注册、查询、选择、统计、排名、归档/恢复、演化与定时调度。
    • 技能数据模型：包含前端元数据（name、description、license、compatibility、metadata、allowed-tools）、正文指令、可选资源（scripts/references/assets）与运行时统计。
    • 技能规范：定义 SKILL.md 的目录结构、YAML frontmatter 字段约束、正文内容建议、可选目录与文件引用规则。
    • 知识处理编译器接口：定义"现有技能打磨"和"新技能发现"的职责，连接记忆系统与技能持久化，支持从 Agent 的肌肉记忆和短期会话记忆中提取高价值经验。
    • 记忆系统：短期记忆（会话上下文）、经验记忆（肌肉记忆）、长期知识（RAG），支撑技能选择与演化。
    • 工具系统：为技能执行提供工具能力，支持安全级别与外部提供者集成。

    架构总览¶

    技能系统通过"规范驱动 + 数据模型 + 管理器 + 记忆系统 + 工具系统"的协同，形成从"技能定义 → 加载/注册 → 选择 → 执行 → 统计 → 演化"的闭环。

    详细组件分析¶

    技能管理器接口与默认实现¶

    • 接口职责
    • 加载：从目录加载 SKILL.md 与可选资源（scripts/references/assets），解析 YAML frontmatter，校验必需字段。
    • 注册：注册技能并进行数据完整性校验。
    • 查询：按名称获取技能，列出元数据。
    • 选择：支持关键词、语义、混合三种模式；混合模式先关键词筛选再语义精挑。
    • 统计与排名：记录执行结果，更新成功率、平均耗时、质量评分、频率评分与综合评分；生成技能排名。
    • 演化与调度：定期执行技能优胜劣汰（归档/软淘汰），支持手动归档/恢复。
    • 关键算法
    • 关键词评分：基于任务词与技能名称/描述的匹配、映射词扩展、历史综合评分加权。
    • 语义选择：构造提示词，调用 LLM，解析返回的技能名称并匹配候选。
    • 综合评分：加权聚合成功率、效率、质量、频率。
    • 并发与一致性：读写分离锁保护活跃/归档技能库，避免竞态。
    • 错误处理：对缺失文件、格式错误、必填字段缺失、LLM 不可用等情况给出明确错误信息。

    依赖关系分析¶

    • 技能管理器依赖
    • YAML 解析：用于 frontmatter 解析
    • LLM 客户端：用于语义选择
    • 工具管理器：用于执行技能（受 allowed-tools 限制）
    • 记忆系统依赖
    • 短期记忆：KV 存储，JSON 序列化
    • 经验记忆：从执行经验中蒸馏的捷径指令
    • 工具系统依赖
    • 工具接口与安全级别
    • 外部提供者注册表（MCP、LangChain 等）

    性能考量¶

    • 选择阶段
    • 关键词评分采用简单排序，适合中小规模技能库；大规模场景可考虑倒排索引或向量化检索
    • topN 控制候选规模，避免 LLM 调用成本过高
    • 统计更新
    • 使用原子/锁保护，避免竞争；批量更新优于频繁细粒度更新
    • I/O 与序列化
    • 记忆持久化采用 JSON，注意大体量会话的压缩与清理
    • 并发
    • 读多写少场景下，优先使用读锁；写操作合并批处理

    故障排查指南¶

    • 常见错误
    • SKILL.md 缺失或格式错误：检查 frontmatter 是否缺失或非法
    • 必填字段为空：name、description、instructions 必须满足长度与内容要求
    • LLM 客户端未配置：语义选择需要 LLM 客户端
    • 技能未找到：确认已注册且名称一致
    • 演化调度未生效：确认已启动定时调度器
    • 排查步骤
    • 校验 SKILL.md 是否符合规范
    • 检查 allowed-tools 是否与实际工具一致
    • 查看统计指标与排名，定位低效技能
    • 检查归档状态，必要时恢复技能

    结论¶

    GoReAct 技能系统以"Agent Skills 规范"为纲，结合"记忆系统 + 工具系统"，实现了从"技能定义 → 加载/注册 → 选择 → 执行 → 统计 → 演化"的完整闭环。通过关键词与语义双重选择、综合评分与演化调度，系统在准确性与效率之间取得平衡；通过知识处理编译器机制，持续将经验转化为可复用的技能，推动 Agent 的自进化能力。新的编译器架构提供了更强的知识提取和技能生成能力，支持从记忆系统中自动发现和优化技能。
