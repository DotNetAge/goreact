    技术栈¶

    简介¶

    本技术栈文档面向 GoReAct 框架的使用者与贡献者，系统梳理框架所采用的 Go 语言版本要求、核心依赖库及其版本信息，并解释选择这些技术的原因与优势。同时覆盖开发工具链、构建系统、测试框架、依赖管理策略、模块化设计与版本兼容性要点，帮助读者快速理解框架的技术基础与生态。

    项目结构与模块化设计¶

    • 框架采用按功能域划分的模块化组织方式，核心模块包括 engine（引擎）、thinker（思考器）、actor（行动者）、observer（观察者）、terminator（终止器）、prompt（提示词）、memory（记忆）、metrics（指标）、log（日志）、tools（工具）、model（模型管理）等。
    • 模块间通过清晰的接口耦合，例如 engine.Reactor 通过函数选项注入 thinker、actor、observer、terminator 等组件；core.PipelineContext 作为跨模块的状态载体，承载会话、追踪、令牌用量、日志与指标等。
    • 该设计保证了可插拔性与可扩展性，便于替换或增强特定认知组件。

    核心依赖与版本信息¶

    • Go 语言版本要求：1.25.1（模块声明明确）
    • 主要外部依赖：
    • github.com/DotNetAge/gochat：用于通用聊天与管道执行（框架核心引擎依赖）
    • go.uber.org/zap：高性能结构化日志库（框架日志适配层）
    • gopkg.in/yaml.v3：YAML 序列化与反序列化（配置与协议）
    • emersion/go-imap、emersion/go-message：IMAP/邮件相关（间接依赖）
    • go.uber.org/multierr：多错误聚合（间接依赖）
    • golang.org/x/text：文本处理（间接依赖）

    选择理由与优势 - gochat：提供统一的管道执行与钩子机制，支撑框架“管线即驱动”的核心理念，便于实现 If/Loop/Sequence 等控制流。 - zap：性能优异、结构化强、支持生产与开发两种配置，满足可观测性与调试需求。 - yaml.v3：稳定、高效、广泛用于配置文件与协议交互，利于跨平台与人类可读性。 - 间接依赖：作为稳定生态的一部分，提供网络协议与文本处理能力，降低重复造轮子成本。

    日志系统与可观测性¶

    • 日志接口抽象：定义统一 Logger 接口与 Field 辅助函数，屏蔽具体实现差异。
    • zap 适配器：提供生产与开发两种默认配置，支持时间编码、字段转换与子 Logger 构建。
    • 可观测性集成：engine.Reactor 在 PipelineContext 中注入 Logger 与 Metrics，贯穿一次会话的全生命周期。

    模型与客户端集成¶

    • 模型管理器：集中注册、校验与创建不同提供商（OpenAI、Anthropic、Ollama）的 LLM 客户端，支持温度、超时、BaseURL 等参数校验。
    • 客户端创建：根据模型配置动态创建对应客户端实例，统一接入框架的执行管线。

    开发工具链与测试框架¶

    • 构建系统：基于 Go Modules，Go 版本要求 1.25.1，使用标准 go 命令进行构建与运行。
    • 测试框架：使用 Go 内置 testing 包，配合示例与单元测试验证核心组件行为（thinker、actor、engine）。
    • 示例与集成测试：examples 目录提供 Ollama 与云端模型的端到端示例，验证框架在真实 LLM 客户端下的运行效果。

    依赖管理与版本兼容性¶

    • 依赖声明：go.mod 明确列出直接依赖与间接依赖，确保可复现构建。
    • 版本策略：优先采用稳定版本，结合 go.mod 的 require/indirect 分组，减少不必要的升级风险。
    • 兼容性建议：
    • 保持 Go 版本不低于 1.25.1，以获得最新语言特性与性能优化。
    • 对第三方库升级需进行回归测试，尤其是 gochat、zap、yaml.v3 等核心依赖。
    • 对 IMAP/邮件相关功能的使用应关注间接依赖的变更，避免破坏性更新。

    架构与数据流概览¶

    • 核心循环：Reactor 将 Thinker、Actor、Observer、Terminator 四个组件组装为统一的执行管线，每轮循环输出 Trace（思维-动作-观察），直至终止条件达成。
    • 状态载体：PipelineContext 携带会话、追踪、令牌用量、日志、指标与附件等信息，贯穿整个执行周期。
    • 可观测性：默认注入 Logger 与 Metrics，支持延迟、错误、Token 使用与资源使用统计。

    结论¶

    GoReAct 以 gochat 为核心引擎，结合 zap 与 yaml.v3 构建高性能、可观测且易扩展的 ReAct 框架。模块化设计使认知组件高度解耦，配合完善的测试与示例，便于在本地与云端模型上快速落地。遵循 go.mod 的依赖约束与版本策略，有助于在长期演进中保持稳定性与兼容性。
