    项目概述¶

    引言¶

    GoReAct 是一个高性能、可编程的 ReAct（推理 + 行动）框架，目标是将人类认知模式自动化。框架倡导“编排模式驱动 (Pattern-Driven Orchestration)”理念，允许通过 Master-Sub 模式拆解复杂任务，并通过 Evolution (动作编译) 机制将推理经验固化为极速执行的 CompiledAction。同时，全链路支持基于 GoRAG 的“语义化匹配”，使智能体能够像资深专家一样进行精准的意图识别与知识召回。

    • 框架愿景：以可编程、可扩展、可观测的方式，构建具备人类灵活性与精确性的智能体系统。
    • 核心能力：三模态记忆系统、通用逻辑管道、思考者作为架构师（Codeword 驱动）、Prompt 协议与安全授权等。

    项目结构¶

    仓库采用按职责分层的组织方式，核心模块围绕“引擎 + 思考者 + 执行者 + 观察者 + 终止器 + 记忆 + 工具 + 提示工程 + 技能”展开，便于替换与扩展。

    核心组件¶

    • 引擎（Reactor）：装配思考者、执行者、观察者与终止器，驱动“思考-行动-观察-检查”的循环，支持钩子与指标埋点。
    • 思考者（Thinker）：负责意图解析、工具发现、提示合成、LLM 推理与结果解析；支持 Codeword 模式（/plan、/specs）与三模态记忆注入。
    • 执行者（Actor）：将决策转化为具体动作，调用工具或技能，负责本地重试与超时边界。
    • 观察者（Observer）：清洗与摘要执行结果，将噪声输出转化为结构化反馈，必要时翻译系统错误。
    • 终止器（Terminator）：在成功、失败阈值、最大步数或资源耗尽时安全中断循环。
    • 记忆（MemoryBank）：三模态记忆（工作记忆、语义记忆、肌肉记忆），支撑上下文与经验沉淀。
    • 工具（Tool/Manager）：统一的工具抽象与管理器，支持安全等级与动态检索。
    • 技能（Skill/Compiler）：从经验中提炼与演化技能，形成可复用的 SOP。

    详细组件分析¶

    引擎（Reactor）与通用逻辑管道¶

    • 职责：装配四类核心组件，构建可插拔的执行管线，支持钩子与指标记录，驱动 ReAct 循环。
    • 关键点：
    • 可通过选项注入自定义 Logger/Metrics/Hook/Thinker/Actor/Observer/Terminator。
    • Run 方法内构建 PipelineContext，组装 Thinker → Actor → Observer → CheckFinish 步骤，循环执行直至终止。
    • 支持在上下文中注入思考流回调，实时打印 LLM 思维过程。

    性能考虑¶

    • 令牌计数与上下文压缩：提示构建器内置令牌估算与滑动窗口压缩策略，降低长上下文开销。
    • 流式思维输出：支持思考流回调，边生成边输出，改善交互体验。
    • 循环节流：ReAct 循环内存在短间隔休眠，避免 CPU 空转。
    • 指标与可观测：内置指标记录与日志注入，便于定位瓶颈与异常。

    故障排查指南¶

    • 初始化失败：若缺少思考者、执行者、观察者或终止器之一，引擎会返回初始化错误。
    • 安全拒绝：当安全钩子在执行前注入错误，执行步骤会直接返回，避免危险操作。
    • 格式错误：思考者在解析 ReAct 输出失败时，会自动注入一次“格式错误反思”的观察，帮助恢复。
    • 日志与指标：通过上下文注入的日志器与指标器，可定位每一步的耗时、令牌用量与错误原因。

    结论¶

    GoReAct 将 ReAct 模型工程化，提供可编程、可扩展的认知闭环：以思考者为架构师，借助三模态记忆与工具生态，通过通用逻辑管道与安全机制，实现从自然语言到可执行计划再到稳定结果的自动化。对于初学者，它提供了清晰的示例与直观的 API；对于资深开发者，它提供了高度解耦的接口与可观测性，便于定制与演进。

    附录¶

    快速开始¶

    • 初始化 Reactor，注入思考者、执行者、观察者与终止器。
    • 使用 /plan 模式将自然语言分解为可执行任务序列。
    • 通过思考流回调实时查看 LLM 的思维过程。
