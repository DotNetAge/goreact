    核心特性¶

    引言¶

    本节概述 GoReAct 框架的目标与定位：以 ReAct 思维链为核心，结合可编程逻辑管道、三模态记忆系统与安全可控的人机协同，实现“思考-行动-观察-终止”的闭环推理与执行引擎。框架强调可扩展性与生产可用性，支持多模型接入、Prompt 协议化构建、以及面向真实业务场景的安全与稳定性保障。

    项目结构¶

    仓库采用按职责分层的组织方式： - engine：反应循环引擎与步骤装配 - thinker：思考与计划编译（含模式解析） - actor：执行动作（含安全钩子） - observer：观察结果标准化 - terminator：终止条件判定 - memory：内存管理（工作记忆、语义记忆、肌肉记忆） - prompt：Prompt 构建与上下文压缩 - tools：工具注册与安全等级 - skill：技能提炼与演化 - steps：步骤封装 - core：核心类型与上下文 - examples：示例与测试

    核心组件¶

    • 反应循环引擎（Reactor）：装配 Thinker/Actor/Observer/Terminator 四大组件，驱动 ReAct 循环；支持钩子注入与上下文选项。
    • 思考者（Thinker）：负责意图与模式解析、工具发现、Prompt 构建、LLM 推理与输出解析；内置 Codeword 模式（/plan、/specs）。
    • 执行者（Actor）：将思考者生成的动作映射到具体工具，执行并返回原始结果；可集成安全钩子进行 HITL 授权。
    • 观察者（Observer）：对执行结果进行清洗、截断与摘要，形成可反馈给 LLM 的标准化观测。
    • 终止器（Terminator）：设定硬性边界（最大步数、完成状态），检测停滞与死循环风险。
    • 记忆银行（MemoryBank）：三模态记忆（Working/Semantic/Muscle）的统一入口，支撑上下文与经验沉淀。
    • 技能提炼（Compiler）：从肌肉记忆与短期会话中提炼高价值经验，生成可复用的技能文档。
    • Prompt 协议：通过 FluentPromptBuilder 提供系统/用户模板、工具描述、历史对话、Few-Shot 示例与上下文压缩能力。
    • 工具与安全：工具安全等级划分与授权策略，结合会话白名单与长期记忆白名单实现 Sudo HITL 安全控制。

    详细组件分析¶

    三模态记忆系统（Working、Semantic、Muscle Memory）¶

    • Working（工作记忆）：短期上下文与会话状态，用于当前任务的即时推理与提示词拼接。
    • Semantic（语义记忆）：基于检索增强的知识库，提供与任务相关的背景知识。
    • Muscle（肌肉记忆）：经验沉淀与操作规范（SOP），用于后续任务的快速复用与优化。
    • 记忆银行接口：统一抽象，便于替换为向量数据库等高级实现；当前默认实现提供基本 KV 存取与持久化。

    依赖分析¶

    • 组件耦合：Reactor 将 Thinker/Actor/Observer/Terminator 解耦为独立步骤，通过 Pipeline 组合；Thinker 依赖 PromptBuilder、TokenCounter、CompressionStrategy、ToolManager、MemoryBank；Actor 依赖 ToolManager 与 SecurityHook；Terminator 依赖 PipelineContext 的状态。
    • 外部依赖：gochat 的 pipeline 作为执行引擎，提供 Hook 机制与 Step 接口；工具与客户端通过接口解耦，便于替换不同模型供应商。

    性能考虑¶

    • 上下文压缩：滑动窗口与优先级策略有效降低 Token 使用，提升吞吐与成本效率。
    • 流式思考：支持思想流输出，改善交互体验与可观测性。
    • 步骤级度量：记录思考延迟等指标，便于性能分析与优化。
    • 最大步数与停滞检测：防止无限循环与资源浪费。

    故障排查指南¶

    • 模式解析失败：当 /plan 解析失败时，自动回退为普通 ReAct 输出；检查 LLM 输出格式与正则匹配。
    • 工具不可用：确认工具已注册且名称一致；检查工具安全等级与授权状态。
    • 上下文超限：启用压缩策略或减少历史轮次；调整最大 Token 预估。
    • HITL 拒绝：检查 Approver 回调是否正确配置；查看会话/长期白名单状态。
    • 终止条件：关注最大步数与停滞警告；必要时放宽边界或优化计划。

    结论¶

    GoReAct 以 ReAct 为骨架，融合三模态记忆、可编程逻辑管道、思考者作为架构师、Prompt 协议与 Sudo HITL 安全，形成一套可扩展、可演进、可审计的智能体框架。通过模块化设计与协议化抽象，开发者可在不牺牲灵活性的前提下，快速构建生产级的推理-行动系统。
