    架构概览¶

    简介¶

    本文件面向希望系统性理解 GoReAct 框架的读者，围绕“思考者（Thinker）、执行器（Actor）、观察器（Observer）、终止器（Terminator）”四大核心组件与“三模态记忆系统”的协同关系，给出整体架构视图与执行流程说明。文档同时解释通用逻辑管道如何支撑 ReAct 循环，覆盖数据流、控制流与组件间依赖，帮助读者建立从宏观到微观的完整认知。

    项目结构¶

    仓库采用按职责分层的组织方式： - 引擎层：引擎装配与循环控制 - 认知层：Thinker/Actor/Observer/Terminator 接口与步骤封装 - 核心类型与上下文：统一的状态容器与消息/轨迹模型 - 记忆与工具：三模态记忆与工具管理 - 技能与编译：技能加载、选择、统计与自进化

    核心组件¶

    • 思考者（Thinker）：负责意图识别、工具/技能编排、检索增强（RAG）与下一步动作决策；可直接标记完成或生成动作。
    • 执行器（Actor）：负责将思考者生成的动作映射为具体工具调用或脚本执行，处理重试与超时边界，产出原始结果。
    • 观察器（Observer）：将执行器的原始输出转化为结构化、降噪后的认知反馈，便于大模型进一步推理。
    • 终止器（Terminator）：在安全边界、成功条件、最大步数等约束下决定是否结束循环。

    详细组件分析¶

    引擎与通用逻辑管道¶

    • Reactor 负责装配 Thinker/Actor/Observer/Terminator 四大组件，构建通用逻辑管道，驱动 ReAct 循环。
    • 管道由四个步骤组成：Thinker、Actor、Observer、CheckFinish；支持钩子注入与错误传播。
    • 循环在每次迭代后检查终止条件，满足则结束；否则短暂休眠继续下一轮。

    R-->>U : 返回最终结果与上下文

    依赖关系分析¶

    • 组件耦合：Reactor 对四组件强依赖，步骤封装将接口与管道解耦；上下文贯穿所有组件，降低横向耦合。
    • 外部依赖：通用逻辑管道来自 gochat，提供统一的 Step/Hook 能力；日志与指标通过核心接口注入。
    • 可扩展点：工具管理器、记忆管理器、技能管理器均暴露接口，便于替换实现。

    性能考量¶

    • 循环节流：每轮循环后短暂休眠，避免 CPU 空转。
    • 上下文压缩：通过 Working Memory 与观察器降噪减少 LLM 上下文开销。
    • 工具与检索：工具与语义检索的延迟直接影响吞吐，建议缓存常用工具与检索结果。
    • 统计与观测：通过指标与日志记录 Token、耗时与成功率，指导调优。

    故障排查指南¶

    • 管道致命错误：当管道执行返回错误时，引擎会记录完成原因与错误信息，便于定位。
    • 终止条件异常：若终止器规则引擎自身报错，应检查策略配置与边界参数。
    • 执行被拒绝：若 Actor 步骤因安全钩子注入错误而提前返回，检查工具安全等级与 HITL 授权流程。
    • 记忆与工具：确认记忆持久化路径与工具注册表是否正确初始化。

    结论¶

    GoReAct 将 ReAct 思想工程化为可编程、可观测、可演化的 Agent 执行框架。通过 Reactor 统一装配 Thinker/Actor/Observer/Terminator，并以通用逻辑管道串联四者，配合三模态记忆与工具/技能体系，形成从“意图—计划—执行—观察—终止”的闭环。该架构既保证了灵活性，又提供了安全与性能的可治理能力，适合在生产环境中落地。
