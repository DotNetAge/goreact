    框架介绍¶

    引言¶

    GoReAct 是一个高性能、可编程的 ReAct（推理 + 行动）智能体框架，面向生产环境设计。它以“编排模式驱动 (Pattern-Driven Orchestration)”为核心哲学，强调意图驱动的动态任务分发与自适应演化执行；通过三模态记忆系统（工作/语义/肌肉）与原生支持的高级编排模式（如 Master-Sub 拆解、Evolution 自适应快径），实现灵活、可控、可演化的认知执行引擎。

    框架全链路对齐 GoRAG 的“语义化匹配”设计，并提供提示词协议化工具箱、安全阻断（HITL）与动作编译（CompiledAction）闭环，帮助开发者从“简单的 ReAct 循环”跃迁到“可编程、可自进化的认知引擎”。

    • 模式驱动 (Pattern-Driven)：内置 Master-Sub（主从式任务拆解）与 Evolution（自适应进化流水线）等工业级编排模式，支持复杂任务的确定性执行。
    • 语义化匹配 (Semantic Matching)：意图、记忆、工具与技能的发现均采用语义向量检索，通过对接 GoRAG 实现最强 RAG 效果。
    • 动作编译 (CompiledAction)：独创的“三态转化”机制，可将成功的推理痕迹编译为带指纹的“肌肉记忆”动作序列，实现零 LLM 推理的极速快径执行。

    项目结构¶

    仓库采用按领域/职责划分的模块化组织方式，核心模块围绕“引擎 + 思考者 + 执行者 + 观察者 + 终止器 + 工具 + 记忆 + 技能 + 提示词”展开，便于替换与扩展。

    核心组件¶

    • 引擎（Reactor）：封装四步循环（思考-行动-观察-终止），通过管道（pipeline）串联各步骤，支持 Hook 注入与上下文选项。
    • 思考者（Thinker）：负责意图与模式解析、工具发现、提示词合成、上下文压缩、LLM 推理与输出解析；支持“/plan”“/specs”等模式与自维护指令。
    • 执行者（Actor）：根据最后一步思考结果执行工具或外部 API，负责并发与超时边界内的本地重试。
    • 观察者（Observer）：接收执行结果并返回给上下文，作为思考者的输入。
    • 终止器（Terminator）：判断是否结束会话，支持自定义终止条件。
    • 工具（Tool）：统一的可执行能力抽象，带安全等级与描述；提供简单管理器与映射工具实现。
    • 记忆（MemoryBank）：三模态记忆（工作/语义/肌肉）的统一抽象，注入提示词以减少幻觉。
    • 技能（Skill）：SOP 数据包，通过注入系统提示与 ReAct 循环实现柔性执行与经验固化。
    • 提示词工具箱：协议化提示词构建、Token 计数与滑动窗口压缩。

    详细组件分析¶

    高级编排模式 (Orchestration Patterns)¶

    • Master-Sub (主从编排)：针对模糊、大型任务，Master 负责任务拆解（Task Decomposition），Sub 负责执行。通过“分而治之”实现复杂逻辑的确定性落地。
    • Evolution (动作编译)：系统通过三态转化（源码态 -> 编译态 -> 执行态）进行自我演化。成功的推理 Trace 被编译为 CompiledAction，通过指纹验证实现毫秒级的快径（Fast-Path）执行。
    • 语义化召回 (Semantic Recall)：基于 GoRAG 的设计，系统不再通过硬编码的名字查找工具或技能，而是通过当前意图进行语义匹配，实现“所想即所得”。

    依赖分析¶

    • 外部依赖：go.mod 明确了 gochat（管道与客户端）、zap（日志）、yaml（技能解析）等关键依赖。
    • 内部耦合：Reactor 对 Thinker/Actor/Observer/Terminator 的依赖清晰；Thinker 依赖工具管理器与记忆体；Agent 作为装配入口连接 Engine 与 Skill 管理器。

    性能考量¶

    • Token 管理：提示词合成阶段内置 Token 计数与滑动窗口压缩，避免上下文溢出；支持“/compress”强制瘦身。
    • 计划编译：通过“/plan”将自然语言计划编译为可执行的 Pipeline 步骤，减少无效尝试与重复计算。
    • 记忆优化：三模态记忆减少重复检索与错误生成；肌肉记忆显著降低 Token 与延迟。
    • 执行边界：Actor 层面的本地重试与超时边界，避免下游失败拖垮整体性能。
    • 日志与指标：统一的日志与指标接口，便于定位瓶颈与优化。

    故障排查指南¶

    • 初始化失败：若缺少 Thinker/Actor/Observer/Terminator 任一组件，Reactor 将返回初始化错误。请检查装配选项。
    • 输出格式错误：Thinker 对 LLM 输出进行解析，若格式不符会自动生成反思轨迹并引导纠正。可检查提示词模板与工具描述。
    • 上下文过长：使用“/compress”或调整压缩策略；必要时启用“/clear”清理历史轨迹。
    • 工具权限问题：高风险工具需 HITL 授权；检查安全钩子与白名单配置。
    • 技能未生效：确认 SkillManager 是否正确记录执行数据并触发演化；检查定时调度器是否运行。

    结论¶

    GoReAct 将经典的 ReAct 模式与先进的模式编排 (Orchestration Pattern) 深度融合，借助语义化匹配、动作编译、三模态记忆与安全阻断机制，构建了可编程、可演化、可扩展的认知执行引擎。对于初学者，它提供了清晰的模式与示例；对于有经验的开发者，它提供了对接 GoRAG 的极致 RAG 能力与高效的执行路径，足以支撑复杂场景下的智能体工业化落地。
