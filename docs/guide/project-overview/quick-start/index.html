    快速开始 (Quick Start)¶

    本指南将帮助你在 5 分钟内运行起第一个具备 Pattern-Driven (模式驱动) 能力的 GoReAct 智能体。

    1. 环境准备¶

    • Go 版本: 1.25.1 或更高
    • LLM 访问: 准备一个 OpenAI 兼容的 API Key（如 OpenAI, DeepSeek, DashScope）或本地运行的 Ollama。

    2. 安装¶

    在你的 Go 项目中引入 GoReAct：

    go get github.com/DotNetAge/goreact

    3. 极速上手示例 (Hello GoReAct)¶

    以下示例展示了如何使用 agent.Builder 快速构建一个能够进行任务规划（/plan）并执行工具的智能体。

    3.1 编写代码 (main.go)¶

    package main

import (
  "context"
  "fmt"
  "log"

  "github.com/DotNetAge/goreact/pkg/agent"
  "github.com/DotNetAge/goreact/pkg/tools/builtin"
)

func main() {
  ctx := context.Background()

  // 1. 创建并配置 Agent 
  // Builder 会自动为你装配 Reactor 引擎、MemoryBank 和 Thinker
  builder := agent.NewBuilder("Assistant")
  builder.WithModel("gpt-4o") // 或 "ollama/llama3"
  builder.WithSystemPrompt("你是一个全能助手，擅长使用工具解决复杂问题。")

  // 注册内置工具
  builder.WithTools(
    &builtin.Calculator{},
    &builtin.DateTime{},
  )

  myAgent, err := builder.Build()
  if err != nil {
    log.Fatalf("构建失败: %v", err)
  }

  // 2. 使用 /plan 模式执行一个复杂任务
  // /plan 会触发 Master-Sub 模式，先拆解任务再执行
  task := "/plan 计算 (123 + 456) * 789 并告诉我现在的时间。"

  fmt.Printf("--- 任务开始 ---\n%s\n\n", task)

  result, err := myAgent.Chat(ctx, "session-quickstart", task)
  if err != nil {
    log.Fatalf("执行出错: %v", err)
  }

  fmt.Printf("\n--- 最终结果 ---\n%s\n", result)
}

    3.2 运行¶

    确保你设置了必要的环境变量（例如 OPENAI_API_KEY），然后运行：

    go run main.go

    4. 进阶：运行内置示例¶

    项目中提供了多个开箱即用的完整示例，涵盖了不同的模型和场景：

    • Qwen (通义千问) 集成: go run examples/qwen_agent/main.go
    • Ollama (本地模型) 演示: go run examples/ollama_agent/main.go
    • 定时任务工具: go run examples/cron_tool/cron_example.go

    5. 核心模式说明¶

    在 GoReAct 中，你可以通过指令前缀直接控制智能体的思考模式：

    指令前缀 激活模式 适用场景
    (无) 标准 ReAct 简单的问答或单步工具调用。
    /plan Master-Sub 涉及多步逻辑、条件分支或需要拆解的复杂任务。
    /specs Specification 需要严格遵守技术规范或记忆约束的场景。
    /fast Evolution 尝试命中“肌肉记忆”快径，实现零 LLM 推理执行。

    6. 常见问题 (FAQ)¶

    • 如何连接本地模型？ 使用 ollama provider。在 Builder 中设置 WithModel("ollama/qwen2.5:7b")，确保本地 Ollama 服务已启动。
    • 工具调用失败怎么办？ 检查工具的 Name() 是否清晰，Description() 是否准确。GoReAct 依赖语义匹配，良好的描述是成功的关键。
    • 想要查看思考过程？ 框架默认输出 Trace。你可以通过注入自定义的 Observer 来捕获每一轮的 Thought 和 Action。
