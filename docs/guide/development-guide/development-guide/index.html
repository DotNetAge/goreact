    开发指南¶

    简介¶

    本指南面向希望在 GoReAct 框架上进行扩展与集成的开发者，系统讲解如何实现自定义工具与思考器、配置安全级别、编写测试与集成测试、使用代理构建器与管理器、以及 Mock 客户端的单元测试方法。文档同时提供流程图与类图帮助理解代码结构，并给出常见问题与调试技巧，兼顾初学者与资深工程师的需求。

    项目结构¶

    GoReAct 采用模块化分层设计，围绕 ReAct 循环（思考-行动-观察-终止）组织核心能力： - engine：反应式引擎与管道（Reactor） - thinker：思考器接口与默认实现（任务编译、提示合成、模式解析） - actor：执行器接口与默认实现（工具路由与执行） - tools：工具接口、安全级别与工具管理器 - agent：智能体装配器与运行时封装 - mock：模拟 LLM 客户端，便于单元测试 - examples：示例与集成测试 - skill：技能管理与演化

    核心组件¶

    • 工具接口与安全级别
    • 工具接口定义名称、描述、安全级别与执行函数；安全级别分为安全、敏感、高风险三档，用于授权与人机协同（HITL）控制。
    • 工具管理器负责工具发现与上下文相关的动态检索。
    • 思考器接口与默认实现
    • 思考器负责意图识别、工具/技能编排、提示合成与模式解析（如 /plan、/specs），并产出下一步动作或完成标记。
    • 执行器与安全钩子
    • 执行器根据最后一步的行动路由到具体工具并执行；安全钩子在高风险操作前触发人机授权流程。
    • 引擎与代理
    • 引擎将思考器、执行器、观察者、终结者组装为 ReAct 循环；代理通过装配器将配置与引擎整合为可运行实体。
    • 技能管理器
    • 支持技能加载、选择、记录执行、统计与演化（归档/恢复）。
    • Mock 客户端
    • 提供稳定的 LLM 响应模拟，便于单元测试与离线验证。

    架构总览¶

    ReAct 循环由引擎驱动，按步骤顺序执行：思考器生成动作或完成标记；执行器根据动作路由到工具并执行；观察者与终结者分别记录与判断终止条件；循环持续直到完成。

    组件详解¶

    自定义工具开发指南¶

    • 实现工具接口
    • 必须实现名称、描述、安全级别与执行函数；描述用于提示构造，安全级别决定授权策略。
    • 可参考内置计算器与 Cron 工具的实现方式。
    • 安全级别设置
    • 安全级别分为安全、敏感、高风险；高风险默认需要人机授权（HITL）。
    • 可通过安全钩子在执行前拦截并授权。
    • 工具注册与检索
    • 使用工具管理器注册工具；默认管理器提供内存注册表与上下文无关的工具列表。
    • 输入解析与健壮性
    • 使用通用输入解析工具将 map 参数解包到严格结构体，提升类型安全与错误提示。
    • 测试策略
    • 单元测试：使用 Mock 客户端与最小化上下文验证工具执行路径与错误分支。
    • 集成测试：在完整 ReAct 循环中验证工具与思考器协作效果。

    依赖关系分析¶

    • 组件耦合
    • 引擎对思考器、执行器、观察者、终结者强依赖；思考器与执行器均依赖工具管理器；思考器可选依赖记忆体。
    • 安全钩子依赖工具管理器与记忆体，拦截执行器步骤。
    • 智能体依赖引擎、技能管理器与记忆体；可作为工具被其他智能体调用。
    • 外部依赖
    • LLM 客户端接口抽象于 gochat；Mock 客户端实现该接口以支持测试。
    • 循环依赖
    • 未见直接循环依赖；思考器与执行器通过管理器解耦。

    性能考量¶

    • 提示压缩与滑动窗口
    • 默认启用滑动窗口压缩策略，必要时可针对强制压缩命令采用更激进策略。
    • Token 计数与用量回调
    • 在 LLM 调用中注入用量回调，累积 Prompt/Completion Token，便于成本与性能监控。
    • 工具执行延迟度量
    • 执行器为每个工具记录耗时指标，辅助定位瓶颈。
    • 循环节流
    • 主循环中存在短间隔休眠，避免 CPU 空转；可根据场景调整。
    • 技能演化与缓存
    • 技能选择支持关键词与语义混合策略；通过统计与演化减少低效技能带来的额外开销。

    故障排查指南¶

    • 常见问题
    • 工具未注册：执行器会记录警告并返回空结果；检查工具管理器注册与名称一致性。
    • 高风险工具未授权：安全钩子会阻塞并返回授权相关错误；确认授权回调配置与白名单。
    • 思考器输出格式错误：默认思考器会插入反思观测并继续循环；检查提示构造与 LLM 输出稳定性。
    • 引擎未装配完整组件：运行时报错要求全部四个组件已配置；检查装配顺序与选项。
    • 调试技巧
    • 使用思考流回调实时查看 LLM 推理片段，便于定位提示与解析问题。
    • 在工具执行函数中设置断点，验证参数解析与分支逻辑。
    • 使用 Mock 客户端固定响应，隔离 LLM 不稳定因素。

    结论¶

    通过本文档，开发者可以基于 GoReAct 的扩展点实现自定义工具与思考器，合理设置安全级别并通过安全钩子保障高风险操作的安全性；利用代理构建器与管理器快速组装可运行的智能体；借助 Mock 客户端与示例测试建立可靠的单元与集成测试体系。框架在性能、可观测性与安全性方面提供了良好基础，适合在生产环境中演进与扩展。

    附录¶

    • 最佳实践
    • 工具实现：保持幂等与可预测；对输入进行严格校验；为高风险工具明确安全级别。
    • 思考器实现：清晰分离模式解析与提示合成；对 LLM 输出进行健壮解析与回退。
    • 测试策略：单元测试覆盖边界条件；集成测试覆盖完整循环；使用 Mock 客户端稳定测试。
    • 安全考虑：高风险工具必须配置授权回调；白名单机制用于降低授权频次；日志与指标记录贯穿全流程。
