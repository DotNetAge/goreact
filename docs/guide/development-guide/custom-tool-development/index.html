    自定义工具开发¶

    简介¶

    本文件面向希望在该系统中开发“自定义工具”的工程师与产品同学，系统性讲解 Tool 接口的实现方法、安全级别与权限控制、工具注册与管理、内置工具示例、测试策略与最佳实践，并提供常见问题与调试技巧。内容兼顾入门友好与工程深度，帮助你在保证安全与性能的前提下快速构建高质量工具。

    项目结构¶

    本项目的“工具体系”位于 pkg/tools 目录，包含： - 工具接口与管理器定义：pkg/tools/tool.go - 内置工具集合：pkg/tools/builtin/*.go - 安全钩子与授权流程：pkg/actor/security_hook.go - 工具使用说明与设计原则：pkg/tools/README.md

    核心组件¶

    • Tool 接口：定义工具的唯一标识、描述、安全级别与执行逻辑。
    • Manager 接口：提供工具检索与可用工具列表能力。
    • SimpleManager：默认的内存工具注册表，适合小规模工具集。
    • MapTool：将函数包装为 Tool 的便捷实现。
    • ExtractInput：将输入 map 安全反序列化为结构体，便于参数校验与类型安全。
    • 安全级别与授权：通过 SecurityLevel 与 SecurityHook 实现高危工具的人机协同授权。

    架构总览¶

    工具在系统中的角色是“原子动作”，由 Thinker 决策、Actor 执行、SecurityHook 在必要时进行人机授权拦截。工具通过 Manager 注册与发现，内置工具提供文件操作、搜索、执行等常用能力。

    详细组件分析¶

    Tool 接口与实现要点¶

    • 必需方法
    • Name() string：工具唯一标识，用于被 Thinker 调用。
    • Description() string：向大模型说明工具用途、输入输出，影响提示词质量。
    • SecurityLevel() SecurityLevel：声明工具风险等级，驱动授权策略。
    • Execute(ctx, input) (any, error)：实际执行逻辑，输入为 JSON 反序列化后的 map。
    • 可选增强
    • 使用 ExtractInput 将 map 安全反序列化到结构体，提升健壮性与可维护性。
    • 对输入参数进行严格校验，避免运行期异常。
    • 对外部资源（文件、进程）进行边界控制与超时设置。

    依赖分析¶

    • 工具接口与管理器
    • Tool 与 Manager 定义解耦，SimpleManager 提供默认实现。
    • ExtractInput 与 MapTool 为工具实现提供便利。
    • 内置工具
    • 共享公共校验与安全检查（common.go）。
    • 文件类工具依赖标准库文件与路径处理。
    • Bash 工具依赖系统命令执行。
    • 安全钩子
    • 依赖工具管理器与内存存储，实现会话与长期白名单。
    • 通过回调函数实现可插拔的人工审批。

    性能考虑¶

    • 工具数量与上下文窗口
    • SimpleManager 适合小规模工具集；大规模场景建议实现基于向量检索的 RAG Manager，按意图召回 Top-K 工具，缓解长上下文瓶颈。
    • I/O 与外部调用
    • 文件读写与目录遍历应限制大小与层级，避免 OOM 与长时间阻塞。
    • Bash 执行应设置超时与输出截断，防止恶意命令耗尽资源。
    • 并发与稳定性
    • 工具内部避免共享可变状态，必要时使用只读副本或本地缓存。
    • 对外部系统调用进行幂等性与重试策略设计（视业务需求）。

    故障排查指南¶

    • 常见问题
    • 工具未被识别：确认已通过 SimpleManager.Register 注册，且 Name() 与调用名一致。
    • 高危工具被阻：检查会话白名单或长期白名单是否生效，或配置审批器回调。
    • 文件相关错误：检查路径清理、大小限制、权限与目标是否为文件而非目录。
    • Bash 执行失败：确认命令参数、环境变量与权限，关注输出与错误字段。
    • 调试技巧
    • 使用 Echo 工具快速验证输入传递链路。
    • 在工具 Execute 中打印关键参数与中间状态（注意生产环境关闭调试日志）。
    • 对涉及外部命令的工具，先在本地 shell 验证命令与参数组合。

    结论¶

    通过 Tool 接口与 Manager 的清晰分离、内置工具的完备示例以及安全钩子的人机协同授权，该系统为自定义工具开发提供了安全、可扩展与易维护的基础。建议在实现新工具时遵循参数校验、错误处理与性能优化的最佳实践，并结合测试策略确保质量与稳定性。

    附录¶

    工具开发步骤清单¶

    • 定义工具结构体并实现 Tool 接口方法。
    • 在 Description 中清晰描述用途、输入输出与约束。
    • 正确设置 SecurityLevel，并在必要时通过安全钩子增加授权。
    • 使用 ExtractInput 与公共校验函数进行参数验证。
    • 编写单元测试覆盖正常与异常路径。
    • 通过 SimpleManager.Register 注册工具并集成到系统。
