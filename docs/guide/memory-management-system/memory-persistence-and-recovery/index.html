    记忆持久化和恢复¶

    简介¶

    本文件围绕 GoReAct 的记忆持久化与恢复机制展开，重点解释 DefaultMemoryManager 的 Persist 与 Load 方法的实现原理、使用场景与最佳实践。内容涵盖： - JSON 文件存储格式、文件命名规则与目录结构设计 - 并发访问控制与内存管理策略 - 错误处理、数据完整性校验与版本兼容性建议 - 性能优化与可靠性保障 - 与整体系统（Agent、Thinker、Compiler）的集成关系

    项目结构¶

    与记忆持久化直接相关的代码位于 pkg/memory 目录，核心为 DefaultMemoryManager；同时，Thinker 通过系统提示词使用记忆内容，Compiler 负责从记忆中提炼经验并反写技能，Agent 管理器负责选择与装配。

    核心组件¶

    • DefaultMemoryManager：提供短期会话状态的内存存储与 JSON 文件持久化/恢复能力，支持并发安全读写。
    • Thinker 系统提示词：在推理阶段将记忆内容注入系统提示，增强输出一致性与准确性。
    • Compiler：从肌肉记忆与短期记忆中提炼高价值经验，驱动技能的打磨与发现。
    • Agent 管理器：负责 Agent 的选择与装配，间接影响记忆的生命周期与使用时机。

    详细组件分析¶

    DefaultMemoryManager：持久化与恢复实现¶

    • 存储结构
    • 内存层：sessions[sessionID] -> map[string]any，按会话隔离状态。
    • 并发控制：读写锁保护，读多写少场景下提升吞吐。
    • 文件存储格式
    • JSON：使用缩进格式保存，便于人工审阅与版本对比。
    • 文件名规则：sessionID+"_memory.json"，确保唯一性与可追溯性。
    • 目录结构：初始化时创建持久化根目录，避免重复创建目录。
    • Persist 流程
    • 读锁保护序列化过程，避免写竞争。
    • 校验会话是否存在，不存在返回错误。
    • 写入文件，权限位 0644。
    • Load 流程
    • 写锁保护加载过程，避免并发覆盖。
    • 若文件不存在，视为正常（首次运行），返回成功。
    • JSON 反序列化失败返回错误。
    • 成功后替换内存中的会话状态。
    • Recall/Update/Compress
    • Recall 返回当前会话状态的 JSON 字符串，作为系统提示词的一部分。
    • Update/Compress 为后续高级记忆（如 RAG/权重衰减）预留扩展点。

    依赖关系分析¶

    • DefaultMemoryManager 依赖标准库的 JSON 编解码、文件系统与同步原语。
    • Thinker 依赖 DefaultMemoryManager 的 Recall 输出，形成“记忆 -> 提示词 -> 推理”的闭环。
    • Compiler 依赖 DefaultMemoryManager 的短期记忆与肌肉记忆接口（抽象），当前实现通过 JSON 文件承载短期状态。
    • Agent 管理器与 DefaultMemoryManager 无直接耦合，但通过 sessionID 串联生命周期。

    性能考量¶

    • 并发与锁粒度
    • 读多写少场景下，读锁显著降低写阻塞；序列化阶段使用读锁即可，写锁仅在加载/持久化时获取。
    • 序列化成本
    • JSON Marshal/Unmarshal 的复杂度与会话状态规模线性相关；建议定期 Compress 清理无效键值，减少体积。
    • I/O 与磁盘
    • 文件权限 0644 保证可读性；建议在高并发写入场景下考虑落盘策略与同步参数。
    • 资源监控
    • 可结合资源监控器观察内存与 GC 行为，评估大规模会话状态对系统的影响。

    故障排查指南¶

    • 常见错误与处理
    • 会话不存在：Persist 返回错误，检查 sessionID 是否正确传递。
    • 文件不存在：Load 返回成功（视为首次运行），属预期行为。
    • JSON 反序列化失败：文件可能被意外修改或损坏，建议备份与回滚。
    • 数据完整性与一致性
    • 建议在写入前进行校验（如校验和），并在失败时保留旧文件副本。
    • 对于并发写入，建议引入文件级锁或原子写入策略。
    • 版本兼容性
    • 当前未内置版本字段，升级时应提供迁移脚本或自动修复逻辑。
    • 日志与可观测性
    • 在关键路径（Persist/Load/Recall）增加日志，记录 sessionID、文件路径与耗时，便于定位问题。

    结论¶

    DefaultMemoryManager 提供了简单可靠的短期会话状态持久化与恢复能力，配合 Thinker 的提示词注入与 Compiler 的经验提炼，构成了从“短期记忆 -> 技能固化”的闭环。通过合理的文件命名、目录设计与并发控制，可在保证可靠性的同时兼顾性能。建议在生产环境中补充版本兼容、原子写入与完整性校验等机制，以进一步提升鲁棒性。

    附录¶

    • 使用建议
    • 为不同 Agent/任务设置独立 sessionID，避免状态交叉。
    • 在推理结束或周期性节点调用 Persist，确保状态落盘。
    • 启动时调用 Load，快速恢复最近会话状态。
    • 定期执行 Compress 清理无效键值，保持 JSON 精简。
    • 最佳实践
    • 将持久化根目录纳入备份策略。
    • 对关键会话启用增量备份与差异比较。
    • 在高并发场景下评估磁盘 IOPS 与锁竞争，必要时拆分目录或采用异步刷盘。
