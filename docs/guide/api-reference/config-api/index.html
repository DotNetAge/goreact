    配置 API¶

    简介¶

    本文件系统化梳理 GoReAct 框架中的“配置 API”，聚焦以下方面： - 引擎配置：如何通过函数式选项装配 Reactor 核心引擎，以及上下文级配置项。 - 组件配置：Thinker、Actor、Observer、Terminator 的可配置项与默认行为。 - 中间件配置：管道钩子与日志/指标注入。 - 参数作用、默认值与推荐设置。 - 配置优先级、覆盖机制与环境变量支持现状。 - 完整配置示例路径与最佳实践。 - 配置验证、错误处理与调试技巧。

    核心组件¶

    本节从“配置 API”的角度，系统梳理各组件的可配置项、默认值与推荐设置。

    • 引擎配置（Reactor）
    • 可注入项
      • WithEngineLogger(core.Logger)：设置全局日志器
      • WithEngineMetrics(core.Metrics)：设置全局指标收集器
      • WithPipelineHook(pipeline.Hook[*core.PipelineContext])：注册管道钩子
      • WithThinker(thinker.Thinker)：注入自定义思考者
      • WithActor(actor.Actor)：注入自定义执行者
      • WithObserver(observer.Observer)：注入自定义观察者
      • WithTerminator(terminator.Terminator)：注入自定义终结者
    • 默认行为
      • 若未显式注入，引擎会使用默认日志器与指标器
      • 构建时按传入顺序应用选项

    • 推荐设置

      • 在生产环境务必注入自定义日志器与指标器，并确保钩子用于审计与可观测性
      • 四大组件必须全部注入，否则初始化失败

    • 上下文配置（PipelineContext）

    • 可注入项
      • WithAttachments(...)：附加多媒体/文件
      • WithLogger(core.Logger)：注入日志器
      • WithMetrics(core.Metrics)：注入指标器
      • WithMaxSteps(int)：设置最大步数以避免无限循环
      • WithThoughtStream(func(string))：注册思维流回调（用于 UI 实时展示）
    • 默认行为
      • MaxSteps 默认为 10；Logger/Metrics 使用默认实现

    • 推荐设置

      • 对高风险任务提高 MaxSteps；对长文本任务启用思维流回调提升交互体验

    • 组件配置（Thinker）

    • 可注入项
      • WithModel(string)：指定模型名
      • WithToolManager(tools.Manager)：注入工具管理器
      • WithMemoryBank(memory.MemoryBank)：注入记忆体
      • WithSystemPrompt(string)：覆盖系统提示模板
    • 默认行为
      • 默认模型名为 gpt-4；默认系统提示来自内置模板；默认工具管理器为空管理器

    • 推荐设置

      • 明确注入 SystemPrompt 以约束行为；为不同场景准备 MemoryBank 与工具集

    • 组件配置（Actor）

    • 可注入项
      • WithToolManager(tools.Manager)：注入工具管理器
    • 默认行为
      • 默认使用空工具管理器作为回退

    • 推荐设置

      • 必须在构建阶段完成工具注册，确保执行阶段可路由到具体工具

    • 组件配置（Observer/Terminator）

    • 接口职责
      • Observer：将原始执行结果标准化为 LLM 可消费的 Observation
      • Terminator：判定是否终止循环（成功、失败、超限等）
    • 默认行为
      • 默认实现位于各自包内（此处不展开具体实现），可通过 WithXxx 注入自定义实现

    • 推荐设置

      • Observer 应统一错误语义化；Terminator 应结合 MaxSteps、错误阈值与业务意图

    • 模型配置（Model）

    • 关键字段
      • Name、Provider、ModelID、APIKey、BaseURL、Temperature、MaxTokens、Timeout、Features、Metadata
    • 默认值
      • Temperature=0.7、MaxTokens=4096、Timeout=30 秒
    • 校验规则
      • Temperature ∈ [0.0, 2.0]
      • MaxTokens、Timeout 必须为正数
    • 推荐设置
      • 根据任务复杂度调整 Temperature 与 MaxTokens；为多模态任务开启相应 Feature

    架构总览¶

    下图展示配置如何贯穿引擎生命周期：从 Reactor 选项注入，到 PipelineContext 的上下文配置，再到组件内部的参数生效。

    participant R as "Reactor

    end
    R-->>Dev : 返回最终上下文与结果

    详细组件分析¶

    引擎配置（Reactor）与优先级¶

    • 选项应用顺序
    • Reactor 构造时按传入顺序依次应用 Option
    • 后应用的选项会覆盖先前设置（例如多次 WithEngineLogger 时，后者覆盖前者）
    • 组件注入要求
    • Thinker、Actor、Observer、Terminator 必须全部注入，否则初始化报错
    • 钩子与中间件
    • WithPipelineHook 可多次调用，按注册顺序执行
    • 日志与指标
    • WithEngineLogger/WithEngineMetrics 注入后，Run 会自动写入 PipelineContext，供组件共享
        flowchart TD
    Start(["开始"]) --> ApplyOpts["应用选项
    WithEngineLogger/WithEngineMetrics/WithPipelineHook/WithThinker/WithActor/WithObserver/WithTerminator"]
    ApplyOpts --> Validate{"组件齐全？"}
    Validate --> |否| Err["返回初始化错误"]
    Validate --> |是| BuildPipe["构建管道并注册钩子"]
    BuildPipe --> Loop["循环执行：Think -> Act -> Observe -> CheckFinish"]
    Loop --> End(["结束"])

    依赖关系分析¶

    • 引擎层
    • Reactor 依赖 Thinker、Actor、Observer、Terminator 接口，通过函数式选项注入
    • Reactor 将引擎级 Logger/Metrics 注入 PipelineContext
    • 组件层
    • Thinker 依赖工具管理器与记忆体；Actor 依赖工具管理器；Observer/Terminator 依赖上下文状态
    • 配置层
    • Model 提供统一的模型配置与校验；Logger/Metrics 由引擎注入

    性能考量¶

    • 控制循环步数
    • 使用 WithMaxSteps 设置合理上限，避免长时间无效迭代
    • 流式输出
    • 通过 WithThoughtStream 实现思维流回调，减少前端等待
    • 指标采集
    • 注入自定义 Metrics 记录 Thinker/Actor/工具延迟，定位瓶颈
    • 模型参数
    • 合理设置 Temperature 与 MaxTokens，平衡质量与成本

    故障排查指南¶

    • 初始化失败：缺少组件
    • 现象：初始化时报错，提示必须配置 Thinker、Actor、Observer、Terminator
    • 处理：确保通过 WithThinker/WithActor/WithObserver/WithTerminator 注入
    • 参考
      • pkg/engine/reactor.go:113-115
    • 工具未注册导致执行失败
    • 现象：Actor 报告工具未注册或不可用
    • 处理：在构建阶段完成工具注册，并通过 WithToolManager 注入
    • 参考
      • pkg/actor/default_actor.go:47-54
    • 模型参数越界
    • 现象：设置 Temperature/MaxTokens/Timeout 时触发校验错误
    • 处理：遵循校验范围与正数约束
    • 参考
      • pkg/model/model.go:107-131
    • 思维流无输出
    • 现象：前端未收到实时更新
    • 处理：确认已通过 WithThoughtStream 注册回调
    • 参考
      • pkg/core/context.go:74-79

    结论¶

    • 配置 API 采用函数式选项与上下文选项相结合的设计，清晰地分离了“引擎级”和“会话级”配置
    • 通过钩子与日志/指标注入，框架提供了强大的可观测性扩展点
    • 组件间通过接口解耦，配置项明确且具备合理的默认值与校验规则
    • 建议在生产环境中统一注入自定义 Logger/Metrics，严格校验模型参数，并为高风险任务设置合适的 MaxSteps

    附录：配置清单与最佳实践¶

    • 引擎配置清单
    • WithEngineLogger(core.Logger)：全局日志器
    • WithEngineMetrics(core.Metrics)：全局指标器
    • WithPipelineHook(pipeline.Hook[*core.PipelineContext])：管道钩子（可多次）
    • WithThinker(thinker.Thinker)：思考者
    • WithActor(actor.Actor)：执行者
    • WithObserver(observer.Observer)：观察者
    • WithTerminator(terminator.Terminator)：终结者

    • 推荐：在生产环境注入自定义 Logger/Metrics，并注册审计/追踪钩子

    • 上下文配置清单

    • WithAttachments(...)：附加文件/多媒体
    • WithLogger(core.Logger)：覆盖引擎日志器
    • WithMetrics(core.Metrics)：覆盖引擎指标器
    • WithMaxSteps(int)：最大步数
    • WithThoughtStream(func(string))：思维流回调

    • 推荐：对长流程开启思维流回调，对高风险任务提高 MaxSteps

    • 组件配置清单

    • Thinker
      • WithModel(string)：模型名
      • WithToolManager(tools.Manager)：工具管理器
      • WithMemoryBank(memory.MemoryBank)：记忆体
      • WithSystemPrompt(string)：系统提示模板
    • Actor
      • WithToolManager(tools.Manager)：工具管理器

    • 推荐：明确注入 SystemPrompt 与 MemoryBank；在构建阶段完成工具注册

    • 模型配置清单

    • 字段：Name、Provider、ModelID、APIKey、BaseURL、Temperature、MaxTokens、Timeout、Features、Metadata
    • 默认：Temperature=0.7、MaxTokens=4096、Timeout=30
    • 校验：Temperature∈[0.0,2.0]，MaxTokens>0，Timeout>0

    • 推荐：根据任务类型调整 Temperature 与 MaxTokens；为多模态任务开启对应 Feature

    • 环境变量支持现状

    • 仓库未发现显式的环境变量解析逻辑

    • 建议：在应用入口处读取环境变量并映射到上述配置项（例如通过 WithXxx(...) 动态装配）
