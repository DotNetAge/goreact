    工具 API¶

    简介¶

    本文件为 Tool API 的全面文档，面向开发者与使用者，系统性说明工具接口定义、生命周期、参数与返回值规范、错误处理机制、工具注册与管理流程，以及内置工具的使用示例与自定义工具开发指南。同时提供测试策略、性能考量与最佳实践建议，帮助在保证安全与可维护性的前提下高效使用与扩展工具体系。

    项目结构¶

    工具相关代码主要分布在以下模块： - 核心接口与默认实现：pkg/tools/tool.go - 内置工具集合：pkg/tools/builtin/*（文件读写、搜索、执行、计算、时间、Cron 等） - 工具提供者与注册表：pkg/tools/provider/provider.go - 示例与测试：examples/crontool/cronexample.go、pkg/tools/builtin/builtin_test.go

    核心组件¶

    • 工具接口 Tool
    • 名称：唯一标识，用于被思考器调用
    • 描述：清晰说明用途、输入输出
    • 安全等级：区分安全、敏感、高风险
    • 执行：接收上下文与原始输入映射，返回任意结果或错误
    • 管理器接口 Manager
    • 按名称检索工具
    • 按意图动态列出相关工具（可扩展为语义匹配）
    • 默认管理器 SimpleManager
    • 基于内存映射的工具注册表
    • 支持批量注册、按名查询、列举全部
    • 工具提供者 Provider 与注册表 Registry
    • 抽象外部工具系统（如 MCP、LangChain Tools、OpenAI Functions）
    • 支持初始化、发现工具、按名获取、健康检查、关闭
    • 统一注册与聚合发现结果

    架构总览¶

    工具系统采用“接口抽象 + 内置实现 + 可插拔提供者”的分层设计： - 上层：思考器根据意图选择工具 - 中层：管理器负责工具发现与检索 - 下层：内置工具或外部提供者执行具体动作 - 安全：通过安全等级与参数校验控制风险

    详细组件分析¶

    工具接口与生命周期¶

    • 生命周期阶段
    • 注册阶段：将工具实例注册到管理器
    • 发现阶段：按意图列出相关工具
    • 调用阶段：根据名称获取工具并执行
    • 错误处理：统一返回错误以便上层决策
    • 参数处理
    • 输入为未解析的 map，工具内部自行校验与解包
    • 提供 ExtractInput 辅助函数进行结构化解析
    • 返回值格式
    • 成功：返回任意结构化的结果映射（如包含布尔标志、统计信息、文本内容等）
    • 失败：返回错误对象，便于上层识别与恢复
    • 安全等级
    • 安全：仅查询类，无副作用（如 Read、Calculator）
    • 敏感：有限写入或受限操作（如 Write）
    • 高风险：任意执行（如 Bash）

    依赖分析¶

    • 组件耦合
    • 内置工具依赖工具接口与通用辅助函数
    • SimpleManager 与 Provider/Registry 互不直接依赖，通过接口解耦
    • 外部依赖
    • 文件系统与命令执行（Read/Write/Bash）
    • 时间解析与格式化（DateTime/Cron）
    • 循环依赖
    • 未发现循环导入；接口与实现分离良好

    性能考虑¶

    • IO 限制
    • Read 设置最大文件大小，避免内存与带宽压力
    • Write 控制写入模式与目录创建成本
    • CPU 与时间复杂度
    • Cron 的 next 迭代受最大迭代次数限制，防止长时间阻塞
    • 字段解析与范围展开应避免冗余计算
    • 并发与上下文
    • Execute 接收 context，便于取消与超时控制
    • 高风险工具（如 Bash）应配合超时与资源配额

    故障排查指南¶

    • 常见错误类型
    • 缺少必填参数：检查调用侧传参
    • 参数类型不符：确认 JSON 解析后的类型
    • 路径安全问题：避免访问受限文件
    • 文件过大/目录误用：遵循 Read/Write 的约束
    • Cron 表达式非法：使用 validate 先行校验
    • 排查步骤
    • 打印输入映射与工具名称，核对参数键名
    • 检查工具安全等级与授权范围
    • 对高风险工具增加超时与输出截断
    • 单元测试参考
    • 参考内置测试用例，覆盖正常路径与异常分支

    结论¶

    Tool API 以简洁的接口与清晰的安全等级划分，提供了从文件操作、搜索、执行到计算与时间处理的完整能力。通过 SimpleManager 与 Provider/Registry 的组合，既能满足快速集成，也能支持复杂生态的工具发现与治理。建议在生产环境中严格遵循参数校验、安全等级与错误处理的最佳实践，并结合测试策略持续保障稳定性与安全性。
