    API 参考¶

    简介¶

    本文件为 GoReAct 的 API 参考，覆盖 Reactor API、Thinker API、Actor API、Tool API、MemoryBank API、SkillManager API 以及 Agent API。文档逐项说明接口职责、方法签名、参数与返回值、典型用法、调用关系与集成模式，并补充配置选项、中间件与钩子、错误处理策略、版本兼容与迁移建议、性能优化与最佳实践。

    项目结构¶

    • 引擎与控制流：engine.Reactor 作为 ReAct 循环的编排器，组装 Thinker、Actor、Observer、Terminator 四大组件并通过管道步骤驱动。
    • 思考与决策：thinker 包提供默认思考器，负责意图解析、工具发现、提示合成、LLM 推理与输出解析。
    • 执行与动作：actor 包提供默认执行器，负责工具路由与执行并将结果写回上下文。
    • 工具系统：tools 包定义工具接口与默认管理器，支持安全等级、输入解析与注册。
    • 记忆体系：memory 包提供三模态记忆抽象与默认内存管理器。
    • 技能管理：skill 包提供技能加载、选择、统计、演化与归档能力。
    • 智能体封装：agent 包提供高层 API，将引擎、记忆与技能整合为可运行的 Agent。
    • 核心类型：core 包提供 Action、Observation、Trace 等核心数据结构。

    核心组件¶

    • Reactor：装配 Thinker/Actor/Observer/Terminator，构建管道并驱动循环；支持注入日志、指标与管道钩子。
    • Thinker：负责意图识别、工具发现、提示合成、LLM 推理与输出解析。
    • Actor：负责工具路由与执行，将结果写回上下文。
    • Tool Manager：工具注册、查询与可用工具列表。
    • MemoryBank：三模态记忆抽象（Working/Semantic/Muscle）。
    • Skill Manager：技能加载、选择、统计、演化与调度。
    • Agent：高层封装，提供 Chat 与作为工具的 Execute。

    架构总览¶

    ReActor 的核心循环为“思考-行动-观察-终止判断”，Reactor 作为引擎编排四组件，通过管道步骤依次执行，支持钩子扩展与中间件式配置。

    详细组件分析¶

    Reactor API¶

    • 职责：装配与运行 ReAct 循环，注入日志、指标与管道钩子，校验依赖并驱动管道执行。
    • 关键方法与签名
    • NewReactor(opts ...Option) *Reactor
    • Run(ctx context.Context, sessionID, input string, customOpts ...core.ContextOption) (*core.PipelineContext, error)
    • WithEngineLogger(core.Logger) Option
    • WithEngineMetrics(core.Metrics) Option
    • WithPipelineHook(pipeline.Hook[*core.PipelineContext]) Option
    • WithThinker(thinker.Thinker) Option
    • WithActor(actor.Actor) Option
    • WithObserver(observer.Observer) Option
    • WithTerminator(terminator.Terminator) Option
    • 参数说明
    • sessionID：会话标识符，贯穿上下文与日志。
    • input：用户意图或任务字符串，可能携带 /plan、/specs 等命令词。
    • customOpts：可选的上下文选项，用于注入 Logger、Metrics 等。
    • hooks：管道钩子，可在每次步骤前后执行自定义逻辑。
    • 返回值
    • PipelineContext：包含 Traces、FinalResult、IsFinished、FinishReason、错误等。
    • error：初始化失败或管道致命错误。
    • 使用示例
    • 参考示例路径：README.md:33-45
    • 调用关系与集成
    • Reactor.Run 内部构建 PipelineContext，组装 Thinker/Actor/Observer/Terminator 步骤，注册钩子后循环执行，直到终止条件满足。
    • 错误处理
    • 未配置四组件时返回初始化失败错误。
    • 管道执行错误时设置完成原因与错误并返回。
    • 性能与并发
    • 循环内存在短延迟，避免忙等；指标与日志按步骤记录，便于观测。

    故障排查指南¶

    • 初始化失败
    • 现象：Reactor.Run 返回“缺少四组件”错误。
    • 处理：确保 WithThinker/WithActor/WithObserver/WithTerminator 均已配置。
    • 工具不存在
    • 现象：Actor 记录告警并写入错误。
    • 处理：确认工具已注册或路径正确。
    • LLM 调用失败
    • 现象：Thinker 返回调用错误。
    • 处理：检查网络、鉴权与模型配置；查看流式事件回调。
    • 输出格式错误
    • 现象：自动反射并附加观察，继续循环。
    • 处理：修正提示模板或工具输出格式。
    • 技能选择失败
    • 现象：语义模式缺少 LLM 客户端或未找到合适技能。
    • 处理：配置 LLM 客户端或调整关键词与描述。
    • 记忆持久化失败
    • 现象：Persist/Load 返回错误。
    • 处理：检查目录权限与磁盘空间。

    结论¶

    GoReAct 提供了清晰的 ReAct 架构与可插拔组件，通过 Reactor 统一编排，Thinker/Actor/Observer/Terminator 各司其职，配合 Tool、MemoryBank、SkillManager 与 Agent 封装，形成从意图到行动再到总结的完整闭环。建议在生产环境中启用日志与指标、合理配置提示压缩与工具安全等级，并根据场景选择合适的记忆与技能选择策略。

    配置 API 与选项¶

    • Reactor 配置
    • WithEngineLogger：注入全局 Logger
    • WithEngineMetrics：注入全局 Metrics
    • WithPipelineHook：注入管道钩子
    • WithThinker/WithActor/WithObserver/WithTerminator：注入组件
    • Thinker 配置
    • WithModel：指定模型名称
    • WithToolManager：注入工具管理器
    • WithMemoryBank：注入记忆银行
    • WithSystemPrompt：自定义系统提示模板
    • SkillManager 配置
    • WithLLMClient：语义匹配所需 LLM 客户端
    • WithSelectionMode：选择模式（KeywordOnly/SemanticOnly/Hybrid）
    • WithTopN：混合模式候选数量
    • Agent 配置
    • NewAgent(name, description, systemPrompt, modelName)：创建智能体模板
    • Chat(ctx, sessionID, task, opts...)：执行任务
    • Execute(ctx, input)：作为工具执行子任务
