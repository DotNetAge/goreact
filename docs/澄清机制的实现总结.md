### 新增文件

**`reactor/ask_user.go`** — `AskUserTool` 打断-恢复机制：
- LLM 可调用的 `ask_user` 工具，接收 `question` 参数
- `Execute()` 内部通过 `select` 阻塞在 channel 上，同时发射 `ClarifyNeeded` 事件通知客户端
- `Respond(answer)` 由外部调用者（WebSocket handler 等）调用，解除阻塞并返回用户答案
- `RespondError(err)` 可注入错误
- `IsWaiting()` / `WaitWithTimeout()` 用于查询状态
- context 取消支持

**`reactor/ask_user_test.go`** — 8 个测试覆盖：Info、阻塞-响应、context 取消、错误注入、超时等待、事件发射

### 数据流

```
LLM Think (DecisionAct, target="ask_user")
  → Act: ask_user.Execute() 阻塞
  → ask_user 内部发射 ClarifyNeeded 事件到 EventBus
  → 客户端收到事件，显示问题给用户
  → 用户回答 → 外部代码调用 askUserTool.Respond(answer)
  → Execute 解除阻塞，返回 "User answered: xxx"
  → ActionResult 事件发射，答案注入 ConversationHistory
  → T-A-O 循环继续，LLM 基于用户回答继续推理
```
