## 实现总结

### 新增文件

**`core/event.go`** — ReactEvent 类型定义：
- 12 种 `ReactEventType`：`ThinkingDelta`, `ThinkingDone`, `ActionStart`, `ActionProgress`, `ActionResult`, `ObservationDone`, `SubtaskSpawned`, `SubtaskCompleted`, `FinalAnswer`, `ClarifyNeeded`, `Error`, `CycleEnd`
- `ReactEvent` 结构体：`SessionID`, `TaskID`, `ParentID`, `Type`, `Data`, `Timestamp`
- 事件载荷类型：`ActionStartData`, `ActionResultData`, `SubtaskInfo`, `SubtaskResult`, `CycleInfo`
- `NewReactEvent()` 工厂函数

**`reactor/eventbus.go`** — InProcessEventBus 实现：
- `EventBus` 接口：`Emit()`, `Subscribe()`, `SubscribeFiltered()`
- `InProcessEventBus`：基于 fan-out channel 的发布/订阅
- 缓冲 256 事件防止阻塞
- 非阻塞发送：慢消费者会丢失事件（不阻塞生产者）
- `Close()` 关闭所有订阅者 channel
- `WithEventBus()` ReactorOption

**`reactor/eventbus_test.go`** — 10 个单元测试

### 修改文件

**`reactor/context.go`** — ReactContext 扩展：
- 新增 `SessionID`, `TaskID`, `ParentID` 身份字段
- 新增 `emitEvent func(ReactEvent)` 回调
- 新增 `EmitEvent()` 便捷方法
- 新增 `NewReactContextWithIDs()` 构造器（子任务用）

**`reactor/reactor.go`** — T-A-O 循环集成事件发射：
- `Reactor` 新增 `eventBus` 字段和 `EventBus()` getter
- `NewReactor` 自动创建 `InProcessEventBus`（可通过 `WithEventBus` 覆盖）
- `Run()` 在关键节点发射事件：Think 完成、Action 开始/结果、Observe 完成、Cycle 结束、Final Answer、Error、ClarifyNeeded

**`reactor/task_tools.go`** — 子任务事件集成：
- `TaskCreateTool` 通过 `WithEventBus(t.reactor.eventBus)` 共享 EventBus
- 子任务启动时发射 `SubtaskSpawned` 事件
- 子任务完成/失败时发射 `SubtaskCompleted` 事件
- 子任务的 ReactContext 自动获取正确的 `TaskID`

### 与 gochat 的关系

```
gochat StreamEvent (LLM token 层) → goreact 消费 → ReactEvent (Agent 业务层) → 客户端
```

goreact 的 `ReactEvent` 是 gochat `StreamEvent` 的**上层消费者和转换器**。当前 `Think()` 使用 `callLLMWithHistory`（非流式）。未来如需流式思维链输出，只需将 `callLLMWithHistory` 改为 `callLLMStream`（使用 `builder.GetStreamFor()`），然后在消费 gochat `StreamEvent` 的同时调用 `ctx.EmitEvent(core.ThinkingDelta, ev.Content)` 即可。

### 客户端使用示例

```go
bus := reactor.NewEventBus()
r := reactor.NewReactor(config, reactor.WithEventBus(bus))

ch, cancel := bus.Subscribe()
go func() {
    for event := range ch {
        switch event.TaskID {
        case "main":
            renderMainPanel(event)
        default:
            renderSubtaskPanel(event)
        }
    }
}()
```
