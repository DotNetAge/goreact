### 3. [TODO:59] Skill 中启用 Cron 工具实现计划任务

**难度**: 高 (4/5)

**现状**: `cron` 工具 (`tools/cron.go`) 目前只是一个 **cron 表达式解析器**，支持 `validate`、`parse`、`next` 三种操作——它只能计算时间，**不能调度执行**。没有任何定时触发、持久化计划、回调 Agent 的能力。

**分析**: 要实现完整的计划任务能力，需要解决三个核心问题：

1. **调度器**: 需要一个后台运行的 cron scheduler，能根据 cron 表达式在指定时间触发任务。可以用 `github.com/robfig/cron/v3` 等库。调度器需要跨进程持久化（否则进程重启后计划丢失）。

2. **唤醒 Agent 机制**: 当计划任务触发时，需要能够创建 ReactContext 并调用 `Reactor.Run()`。这意味着调度器需要持有 Reactor 实例引用，或者通过一个回调接口。需要解决的核心问题：
   - 计划任务的输入（prompt）从哪来？→ 存储在计划记录中
   - 计划任务的上下文（history）是什么？→ 需要一个 session 机制或空历史
   - 计划任务的结果输出到哪？→ 需要一个 sink（回调函数或消息通道）

3. **持久化**: 计划任务的 CRUD 需要持久化存储（文件或数据库），支持增删改查。

**实施**: 需要新建 `scheduler` 模块（约 300-500 行），重构 cron 工具（增加 `schedule`/`unschedule`/`list` 操作），设计计划任务存储接口。估计 1-2 天。
