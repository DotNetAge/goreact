## 三层工具架构

### 1. Task 工具 — 同步内联执行
| 工具 | 用途 |
|------|------|
| `task_create` | 同步执行子任务（同一线程、同一 SystemPrompt/Model） |
| `task_result` | 查询已完成的任务结果 |
| `task_list` | 列出所有任务状态 |

对应 `todo_execute` 的 plan→execute 顺序工作流。

### 2. SubAgent 工具 — 独立异步 Agent
| 工具 | 用途 |
|------|------|
| `subagent` | 启动独立 Agent（自有 SystemPrompt + Model，独立 goroutine） |
| `subagent_result` | 等待单个 Agent 结果 |
| `subagent_list` | 列出所有 Agent 状态 |

### 3. Team 工具 — 多 Agent 团队协作（新增）
| 工具 | 用途 |
|------|------|
| `team_create` | 创建团队 |
| `send_message` | Agent 间 channel 消息通信（支持 message/broadcast/shutdown_request） |
| `receive_messages` | 读取 mailbox 中的消息（支持阻塞等待） |
| `team_status` | 查看团队及成员状态 |
| `wait_team` | **核心** — 阻塞等待所有成员完成，收集结果供主 Agent 最终观察总结 |
| `team_delete` | 清理团队资源 |

## 团队协作流程

```
Main Agent (lead)
  │
  ├─ team_create("refactor-team")
  │
  ├─ subagent(name="researcher", team_name=team_id)  ──► goroutine 1
  │     └─ send_message(to="coder", content="findings")
  │
  ├─ subagent(name="coder", team_name=team_id)       ──► goroutine 2
  │     └─ receive_messages() ← channel 收消息
  │
  ├─ subagent(name="tester", team_name=team_id)      ──► goroutine 3
  │
  ├─ team_status(team_id)    ← 监控进度
  │
  ├─ wait_team(team_id)      ← 阻塞等待全部完成，收集结果
  │
  ├─ [Observe: 综合分析各成员结果]
  │
  └─ team_delete(team_id)    ← 清理
```
