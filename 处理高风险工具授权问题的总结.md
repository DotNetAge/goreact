### 新增文件

**`core/permission.go`** — 权限系统核心类型：
- `PermissionBehavior` — 三态：`allow` / `deny` / `ask`
- `PermissionResult` — 权限决策结果（行为 + 消息 + 可选的参数修改）
- `ToolUseContext` — 工具调用的上下文信息（工具名、参数、安全级别等）
- `ToolPermissionChecker` 接口 — 权限检查器，返回 `PermissionResult`
- `PermissionResponder` 接口 — 响应器，支持 `Respond()`/`RespondError()`/`BlockAndWait()`

**`core/hook.go`** — Hook 系统核心类型：
- `HookEventType` — `pre_tool_use` / `post_tool_use` / `session_start` / `stop`
- `HookResult` — Hook 返回值（可覆盖权限决策、修改输入、阻止执行）
- `PostToolUseContext` — PostHook 的上下文（含执行结果、错误、耗时）
- `Hook` 接口 — 生命周期钩子

**`tools/ask_permission.go`** — 内置权限检查器：
- 实现 `ToolPermissionChecker` + `PermissionResponder`
- 安全策略：`LevelSafe`/`IsReadOnly` → 自动放行；`LevelSensitive`/`LevelHighRisk` → 请求用户授权
- channel 阻塞等待用户响应，支持 context 取消、超时
- 用户可修改参数后放行（`UpdatedInput`）

**`tools/ask_permission_test.go`** — 12 个测试

```
LLM Think (DecisionAct, target="replace")
  → Act: ExecuteTool("replace", params)
```
