## 🎯 清理理由说明

### ❌ 已删除的工具

#### 1. http.go & curl.go (重复设计)
**问题**：
- 功能高度重叠（都是 HTTP 请求）
- 增加维护成本
- LLM 难以选择使用哪个

**替代方案**：
```bash
# 使用 bash 工具执行 curl
bash.Execute(map[string]interface{}{
    "command": "curl -X POST https://api.example.com/data",
})
```

#### 2. echo.go (无价值)
**问题**：
- 仅用于测试，无实际业务价值
- 功能过于简单（原样返回输入）
- 占用工具槽位

**替代方案**：
```bash
# 需要 echo 时使用 bash
bash.Execute(map[string]interface{}{
    "command": "echo 'Hello, World!'",
})
```

#### 3. port.go (低频使用)
**问题**：
- 功能单一（只检查端口占用）
- 使用频率极低
- bash 可轻松替代

**替代方案**：
```bash
# 使用 bash 检查端口
bash.Execute(map[string]interface{}{
    "command": "lsof -i :8080",
})
```

#### 4. filesystem.go (职责不清)
**问题**：
- 包揽所有文件操作（读/写/删除/列表）
- 单个工具过于庞大
- 不符合"单一职责"原则

**替代方案**：
拆分为三个独立工具：
- `Read` - 专注读取
- `Write` - 专注写入
- `Edit` - 专注编辑

## 🔍 清理标准

### 删除标准 ✅
符合以下任一条件的工具予以删除：

1. **功能重复**：与其他工具功能高度重叠
2. **无实际价值**：仅用于测试或过于简单
3. **可轻松替代**：其他工具能更好地完成
4. **低频使用**：使用频率极低且非核心

### 保留标准 ⭐
同时满足以下条件的工具予以保留：

1. **高频使用**：日常开发经常用到
2. **不可替代**：没有更好的替代方案
3. **职责单一**：只做一件事，做到极致
4. **组合性强**：能与其他工具组合使用
