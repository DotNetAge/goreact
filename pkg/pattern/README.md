# ReAct 编排模式 (Orchestration Patterns)

本目录 (`pkg/pattern`) 存放基于 GoReAct 底层基座构建的高级智能体编排模式。

## 1. 架构哲学：机制与策略分离 (Mechanism vs. Policy)

GoReAct 遵循高度解耦的设计哲学：
- **基座 (Mechanism - `pkg/engine`)**：提供纯粹、通用的 ReAct 循环。它只负责“如何运行一步推理和执行”，而不关心任务的业务逻辑或编排策略。
- **模式 (Policy - `pkg/pattern`)**：提供具体的任务编排策略。它定义了如何拆解任务、如何协调多个 Agent、以及如何利用记忆进行自我进化。

开发者可以根据需求选择直接使用底层的 `Reactor`，或者使用本目录下预定义的成熟编排模式。

---

## 2. 预定义模式列表

### 2.1 标准大师-从属模式 (Master-Sub Pattern - `pkg/pattern/mastersub`)
**定位**：现阶段主流的复杂任务处理方案。
- **Master-Agent**：作为“大脑”，负责读取 Skill (SOP)，将复杂任务拆解为一系列原子或复合的子任务 (Tasks)。
- **Sub-Agent**：作为“手脚”，负责执行具体任务。
    - **原子任务**：直接映射到 Tool 调用。
    - **复合任务**：开启一个独立的子 ReAct 循环。
- **核心价值**：通过“任务降维”解决超长上下文和逻辑深度问题。

### 2.2 自适应进化模式 (Evo Pattern - `pkg/pattern/evo`)
**定位**：GoReAct 的独家杀手锏，基于“肌肉记忆”的极致优化。
- **三态转化**：Skill 从“源码态（解释）”进化到“编译态（编译）”，最终进入“执行态（直觉）”。
- **自适应快径 (Adaptive Fast-Path)**：
    - **法官 (Observer)**：比对实际结果 (Actual) 与预期指纹 (Expectation)。若匹配，则跳过 LLM 推理，直接线性执行。
    - **警察 (Thinker)**：当法官判定“不匹配”时，介入调查误差原因，并修正肌肉记忆或路径。
- **核心价值**：极大地降低 Token 消耗，实现毫秒级响应，并赋予 Agent 随使用次数增加而自动进化的能力。

---

## 3. 开发者指引

如果您希望：
1.  **处理线性、确定的 SOP**：请优先尝试 `evo` 模式。
2.  **处理高度动态、未知的复杂工程**：请使用 `mastersub` 模式。
3.  **完全自定义您的 Agent 行为**：请参考本目录下的实现，直接调用 `pkg/engine` 构建您专属的 Pipeline。
