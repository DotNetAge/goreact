## 实现总结

已成功在 goreact 中实现了**三层防御策略**来解决上下文爆炸问题，对应 TODO 第 14 项。

### 新增文件

**`core/limits.go`** — 定义三层防御的核心类型：
- `FileReadingLimits`：文件读取的硬性上限（MaxSizeBytes=256KB, MaxTokens=25000, DefaultLines=500）
- `ToolResultLimits`：工具结果的全局阈值（单工具 50,000 字符，单消息 200,000 字符，压缩触发比率 0.8）
- `PersistedToolResult`：持久化结果的元数据结构
- `ToolResultStorage` 接口：抽象持久化存储

**`core/tool_result_storage.go`** — 磁盘持久化的 `ToolResultStorage` 实现：
- 大结果写入临时目录，返回预览 + 文件路径给 LLM
- LLM 可以通过 `read` 工具读取完整内容
- `PersistedResultTag()` 生成格式化的标签文本

**`core/compact.go`** — 上下文压缩（第三层防御）：
- `ContextCompactor` 接口：LLM 驱动的完整压缩
- `MicroCompact()`：快速非 LLM 压缩（截断大消息）
- `ContextBudget` / `CalculateBudget()`：token 预算状态计算
- `TrimJSONResult()`：智能 JSON 结果修剪
- `DefaultTokenEstimator`：token 估算器

**`reactor/context_defense_test.go`** — 5 个单元测试，全部通过

### 使用方式

```go
// 启用完整三层防御
storage := core.NewDiskToolResultStorage(core.WithMaxResultChars(50000))
reactor := NewReactor(config,
    WithResultStorage(storage),                            // 第二层：结果持久化
    WithResultLimits(core.DefaultToolResultLimits()),       // 第二层：大小阈值
    WithCompactorConfig(core.DefaultCompactorConfig()),     // 第三层：压缩配置
)
```

- Read 工具默认 256KB / 500 行限制
- Grep 工具 100 行 / 50K 字符限制
- Glob 工具 200 结果限制
